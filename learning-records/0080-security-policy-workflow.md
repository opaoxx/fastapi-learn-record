# Learning Record 0080 - Security Policy Workflow

Lesson 0089 added the repository security policy for private vulnerability reporting.

Key artifacts:

- `SECURITY.md`
- `lessons/0089-security-policy-workflow.html`
- `reference/0089-security-policy-workflow-cheatsheet.html`

Core learning:

- Security reports need private, evidence-based handling even in a teaching repository.
- Public issues must not contain exploit details, secrets, private logs, personal data, customer data, or working attack steps.
- Maintainers should classify reports by impact and avoid public disclosure until safe.
- Security fixes still need normal repository verification and focused tests when behavior changes.
