# Learning Record 0076 - GitHub Release Notes and Tags Workflow

Lesson 0085 added the repository release process for publishing course batches.

Key artifacts:

- `.github/RELEASE_PROCESS.md`
- `lessons/0085-github-release-notes-tags-workflow.html`
- `reference/0085-github-release-notes-tags-workflow-cheatsheet.html`

Core learning:

- Use annotated tags for public course batch releases.
- Treat GitHub Releases as remote state, not local Markdown output.
- Put changed lessons, references, workflow changes, verification, known warnings, reading notes, and rollback into every release note.
- Do not silently move a published tag.
