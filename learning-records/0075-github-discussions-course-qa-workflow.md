# Learning Record 0075 - GitHub Discussions Course QA Workflow

## What changed

- Added `.github/DISCUSSIONS.md` to document the course repository's GitHub Discussions workflow.
- Defined recommended discussion categories: Q&A, Study Notes, Ideas, Show and Tell, and Announcements.
- Added decision rules for choosing issues vs discussions.
- Added answer quality checklist and discussion-to-issue conversion flow.
- Added Lesson 0084 and Reference 0084 to explain Discussions as a learning support layer, not a replacement for issue triage or CI gates.

## Why it matters

Issues should stay actionable. Discussions give learners a place for exploratory questions, study notes, ideas, and project sharing while preserving a clean path for real repository work to become issues, project items, pull requests, and verified changes.
