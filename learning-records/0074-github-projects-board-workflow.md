# Learning Record 0074 - GitHub Projects Board Workflow

## What changed

- Added `.github/PROJECT_BOARD.md` to document the recommended GitHub Projects board for the course repository.
- Defined required project fields: Status, Priority, Area, Type, and Lesson.
- Added recommended views: Triage Queue, Ready Work, Course Docs, GitHub Workflow, and Done Review.
- Added Lesson 0083 and Reference 0083 to explain how Projects relate to issue templates, triage labels, pull requests, and CI gates.

## Why it matters

Issue labels classify work, but maintainers still need an execution view. A project board shows what is in intake, ready for contribution, in review, blocked, or done without replacing issue history or merge gates.
