静态前端 fetch：StaticFiles、FormData、任务提交链与分页状态

本节把 FastAPI 后端变成可演示的全栈最小闭环：后端用 StaticFiles 服务 /app/，浏览器用 fetch 上传文件、创建摘要任务、读取任务详情，并通过 GET /tasks envelope 渲染状态筛选和分页历史。

① 本节核心知识框架

静态前端 + API 联调
├─ 静态服务：app.mount("/app", StaticFiles(..., html=True))
├─ 页面入口：first_api/frontend/index.html
├─ 上传链路：FormData -> POST /files/text
├─ 任务链路：POST /tasks/files/{file_id}/summary -> GET /tasks/{task_id}
├─ 历史链路：GET /tasks?status=&limit=&offset=
├─ 错误处理：parseJsonResponse 检查 response.ok
├─ 状态管理：historyState 保存 status/limit/offset/count/itemsLength
└─ 关键边界：fetch 不会因为 4xx/5xx 自动抛异常

② 核心概念底层原理

StaticFiles 与 APIRouter 是两类不同的 ASGI 子应用：前者读取磁盘上的 HTML/CSS/JS 并原样返回，后者把请求分发给 Python 函数执行业务逻辑。把静态前端挂在 /app 下，可以让页面和 API 共享同一个 origin，从而降低本地联调时的 CORS 复杂度。

浏览器端 fetch 是 HTTP 客户端，但它不会把 404/422/500 自动变成异常；只要网络层成功收到响应，Promise 就会 resolve。工程上必须读取 JSON 后检查 response.ok，否则前端会把错误响应当成功数据继续渲染。

FormData 是浏览器构造 multipart/form-data 的标准工具。上传文件时不要手写 Content-Type，因为浏览器必须自动生成包含 boundary 的头部；手写会导致后端无法解析 multipart 边界。

③ 全套代码逐行精读解析

frontend_dir = Path(__file__).parent / "frontend"

app.mount(
    "/app",
    StaticFiles(directory=frontend_dir, html=True),
    name="frontend",
)
  • Path(__file__).parent：定位 first_api 包目录，避免依赖当前命令行工作目录。
  • StaticFiles(directory=frontend_dir)：把 first_api/frontend 作为静态文件根目录。
  • html=True：访问 /app/ 时自动返回 index.html。
  • name="frontend"：给挂载的子应用命名，便于未来 URL 反查或调试。
const historyState = {
  status: "",
  limit: Number(historyLimit.value),
  offset: 0,
  count: 0,
  itemsLength: 0,

前端把列表查询条件和分页结果放入一个稳定状态对象。status 决定筛选条件，limit/offset 决定当前页，count/itemsLength 决定上一页/下一页按钮是否可用。

async function parseJsonResponse(response) {
  const body = await response.json().catch(() => ({}));
  if (!response.ok) {
    throw new Error(`${response.status} ${response.statusText}: ${JSON.stringify(body)}`);
  }
  return body;
}
  • response.json().catch(...)：即使错误响应不是 JSON，也不会让错误处理再次崩溃。
  • response.ok：只在状态码 200-299 时为真。
  • throw new Error(...)：把 HTTP 失败转换为 JS 异常，交给调用处 catch 展示。
const formData = new FormData();
formData.append("file", file);

const uploadResponse = await fetch("/files/text", {
  method: "POST",

append("file", file) 的字段名必须和 FastAPI 参数名 file 对齐；这里只写 X-API-Key，不写 Content-Type。

function buildTaskHistoryUrl() {
  const params = new URLSearchParams({
    limit: String(historyState.limit),
    offset: String(historyState.offset),
  });
  if (historyState.status) {
    params.set("status", historyState.status);
  }
  return `/tasks?${params.toString()}`;
}

URLSearchParams 负责正确编码查询参数，避免手动拼接字符串时漏掉 &、空值或编码问题。

function renderTaskHistory(page) {
  historyState.count = page.count;
  historyState.limit = page.limit;
  historyState.offset = page.offset;
  historyState.itemsLength = page.items.length;
  ...
}

这里明确把后端 envelope 当对象读取。若旧代码继续写 page.map(...)，会因为顶层不是数组而报错。

④ 核心机制运行全流程拆解

浏览器打开 /app/
  -> StaticFiles 返回 index.html
  -> JS 自动 loadTaskHistory()
  -> fetch GET /tasks?limit=5&offset=0
  -> renderTaskHistory(page.items/count/limit/offset)

用户选择 .txt 并点击按钮
  -> new FormData(); append("file", file)
  -> POST /files/text，得到 uploaded.id
  -> POST /tasks/files/{uploaded.id}/summary，得到 createdTask.id
  -> GET /tasks/{createdTask.id}，得到状态和结果
  -> show({uploaded, task})
  -> reload history list

用户切换状态筛选或分页
  -> 修改 historyState.status/limit/offset
  -> buildTaskHistoryUrl()
  -> GET /tasks?status=completed&limit=5&offset=0
  -> 按 envelope 重新渲染列表和按钮

⑥ 开发常见报错+坑点+解决方案

  • 404 /app/：静态目录不存在或挂载路径错误。解决：确认 first_api/frontend/index.html 存在且 app.mount("/app", ...) 执行。
  • 422 field required：FormData 字段名不是 file。解决：使用 formData.append("file", file)。
  • multipart boundary not found：手动设置了错误的 Content-Type。解决：上传文件时让浏览器自动生成。
  • 错误响应显示为成功：未检查 response.ok。解决：统一封装 parseJsonResponse。
  • 分页按钮状态错误：没有使用 count 和当前页长度计算。解决：下一页条件为 offset + itemsLength < count。
  • 401 加载任务历史：前端请求未带或带错 API Key。解决：从输入框读取并放入 X-API-Key。

⑧ 课后记忆习题+标准答案

1. StaticFiles 和 APIRouter 的职责有什么区别？

StaticFiles 读取并返回静态文件；APIRouter 把请求交给 Python 路由函数执行业务逻辑并返回动态响应。

2. fetch 遇到 422 会自动进入 catch 吗？

不会。只要网络层收到响应，fetch Promise 会 resolve；必须检查 response.ok 才能把 HTTP 错误转换为异常。

3. 为什么任务历史渲染要读取 page.items？

因为 GET /tasks 返回的是包含 items/count/limit/offset 的 envelope，顶层不是数组。

⑨ 面试高频真题+解析

题 1：如何用原生前端完成文件上传到异步任务结果展示？

用 FormData 上传文件到文件接口，拿到 file_id；调用任务创建接口拿到 task_id；再查询任务详情或列表，按状态展示 result/error。

题 2：为什么浏览器端不能长期保存后端 API Key？

浏览器代码和存储对用户可见，长期密钥会泄漏。生产环境应使用用户认证、短期 token、服务端代理或细粒度权限。

题 3：前端如何适配后端列表响应从数组变为 envelope？

把渲染输入从数组改为对象：遍历 page.items，用 page.count 判断总量，用 page.limit/page.offset 管理分页状态，并补契约测试。
