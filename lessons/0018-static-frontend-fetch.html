Lesson 0018

静态前端调用 API：从按钮到文件摘要任务

这一节做一个很小但完整的浏览器页面：选择文本文件，点击按钮，页面用 fetch 调用 FastAPI，最后显示任务结果。

0. 本节学习目标

  • 理解静态文件和 API 路由的区别。
  • 用 StaticFiles 把前端页面挂到 /app/。
  • 看懂浏览器里的 fetch、FormData 和请求头。
  • 完成“页面 -> 上传接口 -> 任务接口 -> 查询接口”的调用链。

1. 什么是静态文件

静态文件是服务器直接发送给浏览器的文件，例如 HTML、CSS、JavaScript、图片。它们和 API 路由不一样：

API 路由

执行 Python 函数，读取数据库，返回 JSON。

静态文件

服务器直接把文件内容发给浏览器，浏览器负责渲染和运行 JavaScript。

本节新增的页面放在 first_api/frontend/index.html。

2. 挂载静态前端

from fastapi.staticfiles import StaticFiles

frontend_dir = Path(__file__).parent / "frontend"

app.mount(
    "/app",
    StaticFiles(directory=frontend_dir, html=True),
    name="frontend",
)

mount 的意思是：把一个独立的小应用挂到某个路径下面。这里所有以 /app 开头的请求都交给静态文件服务处理。

启动服务后访问：

http://127.0.0.1:8000/app/

3. 页面调用链

这个前端页面没有引入框架，只用浏览器原生能力：

用户选择 .txt 文件
  -> JS 创建 FormData
  -> fetch POST /files/text
  -> fetch POST /tasks/files/{file_id}/summary
  -> fetch GET /tasks/{task_id}
  -> 页面显示 JSON

这条链路非常重要。以后你把摘要函数换成真实 LLM，或者把文本文件换成 PDF、图片、音频，本质还是这种“输入、提交任务、查询结果”的结构。

4. FormData：浏览器里的 multipart/form-data

const formData = new FormData();
formData.append("file", file);

await fetch("/files/text", {
  method: "POST",
  headers: {"X-API-Key": apiKey},
  body: formData,
});

FormData 会帮浏览器构造 multipart/form-data 请求。这里的字段名必须叫 file，因为后端函数参数也叫 file。

注意：使用 FormData 上传文件时，不要手写 Content-Type。浏览器会自动补上包含 boundary 的正确值。

5. fetch：浏览器里的 HTTP 客户端

const taskResponse = await fetch(`/tasks/files/${uploaded.id}/summary`, {
  method: "POST",
  headers: {"X-API-Key": apiKey},
});

fetch 返回的是 Response 对象，不会因为 404 或 422 自动抛异常。所以页面里封装了一个 parseJsonResponse，先读 JSON，再检查 response.ok。

async function parseJsonResponse(response) {
  const body = await response.json().catch(() => ({}));
  if (!response.ok) {
    throw new Error(`${response.status} ${response.statusText}: ${JSON.stringify(body)}`);
  }
  return body;
}

6. 同源页面为什么还讲 CORS

本节页面运行在 http://127.0.0.1:8000/app/，API 也在 http://127.0.0.1:8000，所以它是同源调用，不会触发跨源限制。

但真实开发里，你很可能用 Vite、React、Vue 或独立静态服务器跑前端，例如 http://127.0.0.1:5500。那时就会回到第 17 课的 CORS 问题。

7. 运行实验

  1. 启动服务：fastapi dev first_api/main.py
  2. 打开 http://127.0.0.1:8000/app/
  3. 选择一个 UTF-8 .txt 文件。
  4. 确认 API Key 是 dev-secret-key。
  5. 点击“上传并生成摘要”。

成功时，你会看到上传文件信息和摘要任务信息。失败时，页面会显示后端返回的状态码和错误内容。

9. 小测：先回忆再展开

1. StaticFiles 和 APIRouter 的职责有什么不同？

StaticFiles 负责发送现成文件，APIRouter 负责把请求交给 Python 路由函数处理。

2. 上传文件时为什么用 FormData？

因为浏览器用它构造 multipart/form-data 请求，正好对应 FastAPI 的 UploadFile。

3. fetch 遇到 422 会自动抛异常吗？

不会。需要检查 response.ok 或状态码，再决定是否抛出错误。

10. 本节总结

本节把后端项目变成了一个可以给别人演示的最小全栈闭环：FastAPI 提供 API，静态页面提供交互，浏览器用 fetch 把两者连起来。这是作品集项目非常重要的一步。
