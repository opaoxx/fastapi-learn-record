Lesson 0020

AI 客户端依赖：先封装 demo 模型，再准备替换真实模型

这一节把预测和摘要逻辑放进一个 DemoAIClient。它现在只是规则函数，但结构已经像真实 AI 服务客户端。

0. 本节学习目标

  • 理解为什么不要在 router 里直接写模型推理逻辑。
  • 看懂 DemoAIClient 如何封装预测和摘要。
  • 理解 Depends(get_ai_client) 如何把模型客户端交给接口。
  • 学会用 dependency_overrides 在测试里替换模型客户端。

1. 真实 AI 接口长什么样

真实 AI 服务通常不是一行函数。它可能需要：

  • 读取模型配置和 API Key。
  • 调用本地模型、远程模型或第三方 API。
  • 处理超时、限流、异常和重试。
  • 记录模型来源、版本和分数。

如果这些都写在 routers/predictions.py 里，路由会很快变成一锅粥。所以我们先把“模型能力”封装成客户端对象。

2. PredictionResult：模型输出的小对象

@dataclass(frozen=True)
class PredictionResult:
    label: PredictionLabel
    score: float
    source: str

dataclass 是 Python 标准库提供的轻量数据类。这里它表示“模型内部输出”。它不是 HTTP 响应模型，HTTP 响应仍然由 PredictionResponse 控制。

3. DemoAIClient：假的模型，真的结构

class DemoAIClient:
    positive_words = {"good", "great", "love", "pleasant", "nice", "happy"}

    def predict_sentiment(self, text: str, mode: str) -> PredictionResult:
        ...

    def summarize(self, text: str) -> str:
        ...

这个类目前还是规则算法：看到积极词就返回 positive，摘要就是取前 12 个单词。但它已经像真实模型客户端一样，对外提供稳定方法。

4. get_ai_client：依赖注入入口

def get_ai_client() -> DemoAIClient:
    return DemoAIClient()

然后在路由里声明：

def predict(
    payload: PredictionRequest,
    ai_client: Annotated[DemoAIClient, Depends(get_ai_client)],
) -> PredictionResponse:
    prediction = ai_client.predict_sentiment(payload.text, payload.mode)
    ...

路由函数不再自己知道“怎么预测”，它只知道“我要一个 ai_client，然后调用它”。这就是依赖注入在 AI 服务里的一个典型用法。

5. 为什么响应模型仍然保留

注意这里有两个模型概念，不要混：

  • DemoAIClient 是“AI 模型客户端”，负责算结果。
  • PredictionResponse 是“API 响应模型”，负责规定返回 JSON 长什么样。

AI 客户端可以换，但 API 合约最好保持稳定。这样前端和调用方不会因为你换模型而被迫改代码。

6. 测试里替换 AI 客户端

class FixedAIClient:
    def predict_sentiment(self, text: str, mode: str) -> PredictionResult:
        return PredictionResult(
            label="positive",
            score=0.99,
            source="fixed-test-client",
        )

def override_ai_client() -> FixedAIClient:
    return FixedAIClient()

app.dependency_overrides[get_ai_client] = override_ai_client

这样测试 /predict 时，就不用依赖真实模型或复杂规则。测试只验证：路由确实调用了依赖注入进来的客户端，并把结果转换成响应。

7. 后台任务里的区别

BackgroundTasks 里运行的是普通 Python 函数，不是路由函数，所以不能直接在后台函数参数里写 Depends。课程代码里采用了更简单的写法：

def process_summary_task(task_id: int) -> None:
    with Session(engine) as session:
        run_summary_task(
            session=session,
            task_id=task_id,
            ai_client=get_ai_client(),
        )

核心思想仍然一样：运行任务时把 AI 客户端作为参数传进去，而不是把摘要规则散落在任务函数里。

8. 课堂练习

请你做两个实验：

  1. 把 DemoAIClient.positive_words 里加入 excellent，调用 /predict 看结果变化。
  2. 在测试里的 FixedAIClient 改成返回 neutral，再跑测试观察断言如何失败。

9. 小测：先回忆再展开

1. 为什么把 AI 逻辑封装成客户端？

因为真实 AI 调用会涉及配置、网络、异常和版本，封装后 router 更稳定，也更容易替换。

2. PredictionResult 和 PredictionResponse 有什么区别？

PredictionResult 是服务内部结果，PredictionResponse 是对外 API 合约。

3. 测试为什么要替换 AI 客户端？

为了让测试稳定、快速、可控，不依赖真实模型输出或外部服务。

10. 本节总结

本节完成了一个重要转折：项目里的 AI 逻辑不再散落在路由和 worker 中，而是进入 DemoAIClient。这让后续接真实模型时，只需要替换客户端实现，尽量不动 API 合约。
