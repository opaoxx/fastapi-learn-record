② 核心概念底层原理

GitHub 仓库首页默认渲染根目录 README.md。对课程型项目来说，README 是读者的第一接触点；如果它只写项目简介，不给学习路线和运行路径，新手会迷失在目录树里。

README 的核心价值是降低首次进入成本。它应该回答四个问题：这是什么项目、从哪里开始学、如何运行、如何验证没有改坏。对于本课程，还要告诉读者 provider metrics 与 runbook 训练包在哪里。

本节给 README 加文档契约测试，是为了避免未来改动把关键入口删掉。README 本质上也是公共接口：它服务人类读者，测试保护的是“可发现性”和“学习路径稳定性”。

文档层级面向对象职责
README.md第一次打开仓库的人总入口、学习路线、运行测试
index.html课程学习者课程和速查表目录
first_api/README.md运行项目的人后端项目结构和启动细节
reference/provider-metrics-index.html排障训练读者provider metrics 文档入口

⑧ 课后记忆习题+标准答案

1. 为什么 GitHub 课程仓库必须重视根 README？

GitHub 默认展示根 README，它决定新读者能否快速理解项目定位、学习入口、运行方式和测试方式。

2. 为什么 README 要链接 first_api/README.md？

根 README 负责总导航，子项目 README 负责具体运行和结构细节，二者职责不同。

3. 为什么要测试 README 内容？

README 是公共入口，关键链接和命令被删会影响学习者。测试能保护这些文档契约。

4. 为什么 README 要说明 GitHub HTML 阅读限制？

GitHub 直接打开 HTML 通常显示源码，读者需要克隆到本地后用浏览器打开 index.html。

5. 根 README 为什么不应写成超长教程？

它应提供全局地图和入口，具体知识细节应留给 lesson 和 reference 页面。

⑨ 面试高频真题+解析

面试题 1：README 在工程项目中承担什么职责？

README 是项目的默认入口，承担定位说明、快速开始、运行测试、目录导航、贡献或发布入口等职责。

面试题 2：为什么文档也需要测试？

关键文档是团队协作契约。测试可以防止入口链接、运行命令、API 文档和学习路线在重构中悄悄失效。

面试题 3：根 README 和模块 README 如何分工？

根 README 负责项目总览和导航，模块 README 负责局部运行方式、模块结构和细节约束。

面试题 4：如何让 GitHub 教程对新手友好？

提供明确学习路线、最短运行路径、测试命令、常见阅读坑点、核心文档入口和稳定链接。
