② 核心概念底层原理

本仓库的课程不是由框架编译出来的单页应用，而是一组普通静态 HTML 文件。浏览器读取 index.html，再通过相对链接进入 lessons/ 和 reference/，页面样式由 assets/course.css 提供。

GitHub 文件视图不是静态站点托管。你在 GitHub 网页上点击 HTML 文件，通常看到的是源码预览，而不是课程页面。GitHub Pages 才是 GitHub 提供的静态站点发布层，它会通过 HTTP 服务渲染 HTML、CSS 和相对路径。

相对路径是本节核心。根目录 index.html 使用 assets/course.css，课程页使用 ../assets/course.css，速查页也使用 ../assets/course.css。如果 Pages 只发布某个子目录，路径就会断裂。

模式入口适合场景典型限制
本地文件index.html离线复习、下载后阅读浏览器 file path 行为与 HTTP 略有差异
FastAPI app/app/测试后端和小前端不是课程文档主站
GitHub file view仓库文件页面看源码和 diffHTML 常显示源码
GitHub PagesPages URL公开分享课程站点需要远程配置或工作流

GitHub Pages 是远程状态。提交 .github/PAGES.md 只记录配置契约，不会自动开启 Pages。真正启用需要在仓库 Settings 中选择发布源，或使用 Pages 工作流。

③ 全套代码逐行精读解析

# GitHub Pages and Offline HTML Workflow

标题定义文档职责：解释 Pages 与离线 HTML 阅读，而不是替代 README 或发布脚本。放在 .github/ 下，说明它属于 GitHub 仓库治理文档。

This repository stores course lessons and quick references as static HTML files.

首句定位技术形态。课程页和速查表都是静态 HTML，没有构建步骤、没有前端框架路由、没有运行时打包。

The course can be read offline by opening index.html from a local clone.

这行说明最稳定阅读方式：克隆或下载仓库后，直接从本地打开根目录 index.html。这满足用户要求的离线复习阅读。

GitHub Pages is an optional remote publishing layer; enabling it is repository remote state and is not created by committing this file.

该句防止误解。文档只描述流程；Pages 开关、部署状态和 URL 都是 GitHub 远程配置。

## Reading Modes

Reading Modes 对比四种入口。新手常把 GitHub 文件视图误认为发布站点，因此需要明确区分。

| Local file | index.html | Offline review after cloning or downloading the repository. |

本地文件模式不依赖网络和后端服务，最适合期末复习。缺点是某些浏览器对 file URL 的安全策略与 HTTP 不完全一致。

| FastAPI app | http://127.0.0.1:8000/app/ | Testing the runnable backend and tiny frontend. |

FastAPI app 是可运行项目入口，不是课程文档入口。它用于上传文件、任务历史和 API 调用实验。

| GitHub file view | Repository HTML file page | Inspecting source in GitHub. |

GitHub 文件页适合看源码，不适合当课程站点。直接点 HTML 常看到源代码，是正常行为。

| GitHub Pages | Published site URL | Sharing rendered static course pages. |

Pages 模式适合分享给其他学习者。它通过 HTTP 发布静态文件，使浏览器按网页方式渲染课程。

## Recommended Pages Source

推荐发布源是 main 分支根目录。因为本仓库所有相对路径都以仓库根目录为基准设计。

Branch: main
Folder: /
Entry: index.html

这三行是 Pages 配置核心。若发布 /docs 子目录，现有 assets/、lessons/、reference/ 链接都需要重排。

## Required Static Assets

Required Static Assets 列出完整站点必需文件。只发布单个 lesson 会导致 CSS 和导航链接缺失。

index.html
assets/course.css
all files under lessons/
all files under reference/

这些文件共同构成离线课程站点。index.html 是入口，CSS 是样式，课程和速查是内容主体。

Do not publish only a single lesson file; lesson navigation depends on shared relative paths.

这句解释为什么不能把某节课孤立上传。课程页的上一课、下一课、速查和目录链接都依赖目录结构。

## Pre-Publish Checklist

发布前检查把“站点能打开”转化为可验证流程。至少要跑文档契约、全量测试和空白检查。

.\.venv\Scripts\python.exe -m pytest tests\test_course_docs_contract.py -q
.\.venv\Scripts\python.exe -m pytest -q
git diff --check

docs contract 保护导航和固定课程结构；full pytest 防止示例项目被破坏；diff check 防止空白错误进入发布。

README.md explains that GitHub file view may show raw HTML source.

README 必须提前解释 GitHub 文件视图限制，否则新手会以为课程页面坏了。

## Common Failure Modes

故障表面向发布后排查。静态站点最常见问题不是运行时异常，而是路径、资源、部署源和缓存。

HTML source is shown instead of a page.

这通常不是代码错误，而是用户打开了 GitHub 文件视图。解决方式是打开本地文件或 Pages URL。

Page has no styling.

样式丢失通常表示 assets/course.css 未发布，或相对路径因发布目录变化而断裂。

Lesson link returns 404.

404 表示索引链接指向了未提交文件、文件名不一致，或 Pages 部署源不包含该路径。

## Production Notes

生产文档站点通常会使用静态站点生成器、链接检查、搜索、版本化、CDN 和预览部署。本课程故意保留 plain HTML，便于新手理解路径和部署。

⑧ 课后记忆习题+标准答案

1. 为什么 GitHub 文件视图不等于 GitHub Pages？

文件视图用于查看源码和 diff，HTML 常显示源代码；GitHub Pages 是静态站点托管层，会通过 HTTP 渲染 HTML 和 CSS。

2. 本仓库推荐从哪里发布 Pages？

推荐从 main 分支根目录 / 发布，入口为 index.html，这样 assets、lessons、reference 的相对路径与离线阅读一致。

3. 发布静态课程站点必须包含哪些资产？

必须包含 index.html、assets/course.css、所有 lessons/ 文件和所有 reference/ 文件。

4. 页面样式丢失通常是什么原因？

通常是 assets/course.css 没发布，或发布目录改变导致相对路径断裂。

5. 为什么发布前要跑 docs contract？

docs contract 能保护课程固定结构、公开入口和导航链接，防止发布后出现缺课、断链或速查页漏接。

⑨ 面试高频真题+解析

面试题 1：静态站点托管和后端服务托管有什么区别？

静态站点托管只分发 HTML/CSS/JS/图片等文件，不执行服务器业务逻辑；后端服务托管运行进程，处理请求、数据库、鉴权和状态。

面试题 2：为什么相对路径对离线 HTML 重要？

相对路径让本地文件和 Pages HTTP 站点共享同一目录结构，不依赖固定域名，便于克隆后离线阅读。

面试题 3：GitHub Pages 发布常见 404 如何排查？

检查文件是否提交、路径大小写是否一致、Pages 发布源是否包含该目录、index 链接是否正确、部署是否刷新。

面试题 4：生产文档站点通常比 plain HTML 多什么？

通常有静态站点生成器、搜索、版本化、链接检查、预览部署、CDN、analytics、SEO、accessibility 和回滚策略。

面试题 5：为什么本课程仍选择 plain HTML？

plain HTML 降低依赖和构建门槛，新手能直接理解文件结构、相对路径、CSS 引用和离线阅读机制。
