② 核心概念底层原理

静态 HTML 课程站点没有前端路由器，也没有服务端动态生成目录。用户能否连续学习，完全取决于每个 <a href> 是否指向真实文件。因此链接不是装饰元素，而是课程站点的公共接口。

链接审计的底层问题是路径解析。浏览器解析 href 时，会以当前 HTML 文件所在目录作为基准目录。根目录 index.html 指向 lessons/0094...；课程页位于 lessons/，返回目录要写 ../index.html；速查页位于 reference/，回课程要写 ../lessons/...。

链接检查必须区分仓库内链接和仓库外链接。仓库内相对路径可以用文件系统验证；https:// 外部地址需要网络可用性、重定向和状态码检查，不适合混进离线课程契约测试；#fragment 属于页内定位，和文件存在性不是同一类问题。

链接类型示例是否纳入本节测试原因
本地相对文件../index.html是可用文件系统确定存在性
同目录文件./0093-...是相邻导航依赖该路径
外部 HTTPhttps://example.com否受网络、重定向、限流影响
邮件链接mailto:...否不是文件路径
页内锚点#section否属于 DOM 锚点，不是文件存在性

生产级文档站点通常使用 link checker、crawler、redirect map 和预览部署。本课程选择把第一层问题压到最小：只验证仓库内静态文件链接。这一层足够保护新手最容易遇到的 404、速查缺失、上一课断链和目录漏项。

③ 全套代码逐行精读解析

from urllib.parse import urlsplit

引入 urlsplit 用于拆分 href。链接可能带 ?query 或 #fragment，文件系统检查只应使用 path 部分。

def iter_static_course_html_pages() -> list[Path]:

该函数集中定义静态课程页扫描范围。范围写成函数，避免多个测试各自复制 index.html、lessons、reference 的扫描逻辑。

return [Path("index.html"), *Path("lessons").glob("*.html"), *Path("reference").glob("*.html")]

第一项是课程总入口；后两项收集所有课程页和速查页。若未来新增 lesson/reference，测试会自动纳入，不需要手动补路径。

def is_repository_local_href(href: str) -> bool:

该函数判断链接是否应该由仓库文件系统负责。抽成函数可以避免在测试主体里堆条件判断。

return not href.startswith(("http://", "https://", "mailto:", "#"))

排除外部 HTTP、邮件和纯页内锚点。若不排除外部链接，离线测试会被网络状态污染；若不排除 #，会把页内锚点误判成文件路径。

parsed_path = urlsplit(href).path

只保留 URL path。例如 lesson.html#section 的文件部分是 lesson.html，fragment 不参与文件存在判断。

target = (page.parent / parsed_path).resolve()

以当前页面所在目录解析相对路径。课程页中的 ../index.html 与目录页中的 lessons/... 基准目录不同，必须按页面位置解析。

target.relative_to(root)

验证目标仍在仓库根目录内。否则 ../../outside.html 这类路径可能跳出仓库，虽然在某些本地机器存在，但不属于课程站点。

assert not broken_links

最终断言没有断链。失败信息保留来源页面、原始 href 和解析目标，便于维护者直接定位漏改位置。

# Static Link and Navigation Audit

.github/LINK_AUDIT.md 的标题定义文档职责：它不是最终发布总审计，而是专门管理静态链接、导航模式和路径规则。

The course is a plain HTML learning site, so navigation quality is part of the public API.

该句给出设计原则：静态课程的导航质量等同于公共 API 质量。断链不是排版小问题，而是学习路径中断。

The audit checks local relative links only.

这行明确测试边界。边界越清楚，测试越稳定；不稳定的外部网络检查应放到单独的发布工具，而不是基础文档契约测试。

④ 核心机制运行全流程拆解

pytest starts
  -> load tests/test_course_docs_contract.py
  -> collect static HTML pages:
       index.html
       lessons/*.html
       reference/*.html
  -> parse each page with HTMLParser
  -> collect every a[href]
  -> skip external/http/mail/fragment links
  -> strip query and fragment from local href
  -> resolve target relative to current page directory
  -> confirm target stays inside repository root
  -> confirm target file exists
  -> fail with source page + href + target when broken

这条链路把浏览器点击行为转换为文件系统验证。它不启动服务、不访问网络、不依赖 GitHub Pages，因此适合每次课程更新后快速执行。

⑧ 课后记忆习题+标准答案

1. 为什么静态 HTML 课程必须做链接契约测试？

静态课程没有运行时路由兜底，所有学习路径都依赖显式 href。链接契约测试能在发布前发现缺课、缺速查、目录漏项和相邻导航断裂。

2. 本节链接测试为什么只检查仓库内相对链接？

仓库内相对链接可通过文件系统稳定验证；外部链接受网络、重定向、限流影响，混进基础测试会降低确定性。

3. 为什么解析链接时要以当前页面目录为基准？

浏览器解析相对路径时使用当前文档所在目录。index、lesson、reference 位于不同层级，同一个 href 写法在不同目录下含义不同。

4. 为什么不能在课程 HTML 中写本机绝对路径？

本机绝对路径只在作者电脑存在，克隆到其他机器或发布到 GitHub Pages 后会失效，破坏离线学习和公开分享。

5. 新增一节课时最容易漏掉哪些链接？

最容易漏掉上一课的下一课链接、上一张速查的下一张速查链接、index 的课程入口、index 的速查入口，以及新课到速查页的链接。

⑨ 面试高频真题+解析

面试题 1：如何给静态文档站设计断链检测？

先划分内部链接和外部链接。内部链接用 HTML parser 收集 href，以来源页面目录解析路径并检查文件存在；外部链接用独立 crawler 或发布任务检查，避免污染基础单元测试。

面试题 2：为什么静态站点更依赖构建期或测试期链接检查？

静态站点发布后不会动态查询路由表，也没有服务端兜底生成页面。文件缺失或路径错误会直接变成 404，因此要在发布前发现。

面试题 3：相对路径和 base path 冲突时如何处理？

教学离线站优先保持相对路径；生产站如果引入 base path，需要统一配置生成器、资源路径和路由重写，并用预览部署验证。

面试题 4：为什么不要用正则解析 HTML 链接？

HTML 属性顺序、空白、引号和实体都可能变化。标准 HTMLParser 至少能按标签和属性语义提取 href，比正则扫文本更稳。

面试题 5：断链测试失败信息应该包含什么？

至少包含来源页面、原始 href、解析后的目标路径和失败原因。这样维护者能直接定位是写错 href、漏提交文件还是跳出仓库。
