Depends 依赖注入：FastAPI 的请求级自动装配机制

依赖注入是 FastAPI 从 demo 走向工程项目的关键机制。查询参数封装、数据库 Session、API Key 鉴权、配置读取、AI client、provider HTTP client，本质都可以通过 Depends 在请求进入业务函数前准备好。

① 本节核心知识框架

FastAPI Depends
├─ 解决问题：公共参数、公共资源、公共校验重复编写
├─ 依赖类型
│  ├─ 函数依赖：get_session / get_settings / get_provider_http_client
│  ├─ 类依赖：ItemFilters / Pagination
│  └─ 安全依赖：require_api_key
├─ 执行时机：路径函数执行前
├─ 注入结果：依赖返回值成为函数参数
├─ 可组合性：依赖内部还可以声明依赖
├─ 可测试性：app.dependency_overrides 替换依赖
└─ 当前项目落点：list_items(filters, pagination, session)

② 核心概念底层原理

依赖注入不是把对象“全局塞进去”，而是请求级求值。FastAPI 分析路径函数签名，发现 Depends(...) 后，先执行依赖可调用对象，再把结果作为参数传给路径函数。依赖失败时，路径函数不会执行。

依赖图是有向无环调用链。一个路径函数可以依赖多个对象，依赖内部也可以继续依赖其他对象。FastAPI 会构建依赖图并按顺序解析。循环依赖会导致设计错误，应避免。

类依赖的本质是调用构造函数。ItemFilters 是类，但类在 Python 中可调用；FastAPI 会分析 __init__ 参数，从 query 中解析并创建实例。

可测试性来自可替换边界。测试里用 app.dependency_overrides[get_session] = override_get_session，可以把真实数据库换成内存数据库。这是依赖注入比硬编码全局对象更适合工程测试的核心原因。

③ 全套代码逐行精读解析

class ItemFilters:
    def __init__(
        self,
        q: Annotated[str | None, Query(max_length=40)] = None,
        category: ItemCategory | None = None,
        max_price: Annotated[float | None, Query(ge=0)] = None,
        in_stock: bool | None = None,
    ) -> None:
        self.q = q
        self.category = category
        self.max_price = max_price
        self.in_stock = in_stock
  • class ItemFilters：定义筛选参数对象。若散落在多个路径函数里，维护成本会升高。
  • q: Annotated[str | None, Query(max_length=40)] = None：可选搜索词，最长 40。没有上限时，搜索参数可能过长并污染日志。
  • category: ItemCategory | None = None：可选分类筛选，复用 Literal 枚举。
  • max_price: Annotated[float | None, Query(ge=0)] = None：可选价格上限，禁止负数。
  • in_stock: bool | None = None：可选库存筛选；None 表示不按库存过滤。
  • self.q = q 等赋值：把解析后的 query 参数保存为对象属性，供路径函数使用。
class Pagination:
    def __init__(
        self,
        skip: Annotated[int, Query(ge=0)] = 0,
        limit: Annotated[int, Query(ge=1, le=50)] = 10,
    ) -> None:
        self.skip = skip
        self.limit = limit
  • skip：跳过前 N 条，等价 offset。必须 ge=0，否则切片语义会混乱。
  • limit：本页最多返回条数。必须限制上限，否则调用者可一次拉取大量数据。
  • self.skip/self.limit：统一分页参数对象，后续多个列表接口可复用。
def list_items(
    filters: Annotated[ItemFilters, Depends(ItemFilters)],
    pagination: Annotated[Pagination, Depends(Pagination)],
    session: Annotated[Session, Depends(get_session)],
) -> list[Item]:
  • filters：声明“我需要筛选条件对象”，FastAPI 负责创建。
  • Depends(ItemFilters)：依赖目标是类本身，执行结果是实例。
  • pagination：声明分页对象，避免 skip/limit 在签名中到处重复。
  • session：声明数据库 Session。若硬编码全局 session，会出现并发、生命周期和测试隔离问题。
def get_session() -> Generator[Session, None, None]:
    with Session(engine) as session:
        yield session
  • Generator[Session, None, None]：说明这是 yield 依赖，产出一个 Session。
  • with Session(engine)：请求处理期间打开会话，请求结束后自动关闭。
  • yield session：把 session 注入给路径函数；yield 后面的清理逻辑由上下文管理器完成。

④ 核心机制运行全流程拆解

GET /items?category=course&skip=0&limit=10
  -> 匹配 list_items
  -> 解析 filters 依赖
       ItemFilters.__init__(q=None, category="course", max_price=None, in_stock=None)
  -> 解析 pagination 依赖
       Pagination.__init__(skip=0, limit=10)
  -> 解析 session 依赖
       get_session() yield Session
  -> 执行 list_items(filters, pagination, session)
  -> 响应完成
  -> get_session 的 with 块退出，Session 关闭

如果 limit=100，失败发生在 Pagination 依赖解析阶段；list_items 不会执行。这说明依赖既能“准备资源”，也能“提前拦截非法请求”。

⑤ 重难点、易混淆点对比辨析

  • Depends vs 直接调用：Depends(get_session) 让 FastAPI 管理生命周期；get_session() 直接调用只会得到生成器，语义错误。
  • 依赖返回值 vs 依赖函数本身：路径函数收到的是依赖执行后的返回值或 yield 值，不是 Depends 对象。
  • 类依赖 vs Pydantic 模型：类依赖适合组织 query/公共逻辑；Pydantic 模型更适合请求体 schema。
  • 全局对象 vs 依赖：全局对象难测试、生命周期不清；依赖可按请求创建、可覆盖、可清理。

⑥ 开发常见报错+坑点+解决方案

  • TypeError: 'Depends' object is not ...：在普通函数里直接使用 Depends，但函数不是 FastAPI 路径/依赖上下文。解决：只在 FastAPI 管理的签名中声明。
  • limit=0 返回 422：违反 Query(ge=1)。解决：传 1 到 50。
  • 测试污染真实数据库：没有覆盖 get_session。解决：使用 app.dependency_overrides 注入测试 session。
  • 忘记清理 dependency overrides：后续测试被污染。解决：在 finally 中 app.dependency_overrides.clear()。
  • yield 依赖泄漏资源：手写打开资源但不关闭。解决：使用 with/try finally 包裹 yield。

⑦ 生产环境 vs 教学环境 核心差异

  • 数据库 Session：教学 SQLite session 简单；生产要考虑连接池、事务回滚、超时、读写分离。
  • 鉴权依赖：教学 API Key；生产通常解析 JWT/OAuth2、加载用户、校验权限、记录审计。
  • 外部客户端：教学 lifespan 创建 httpx client；生产要加连接池参数、重试、熔断、指标、trace。
  • 测试覆盖：教学覆盖关键依赖替换；生产还要覆盖权限矩阵、事务失败、依赖超时和清理行为。

⑧ 课后记忆习题+标准答案

1. Depends 的核心作用是什么？

声明路径函数需要的公共对象或逻辑，由 FastAPI 在执行路径函数前解析并注入。

2. 类为什么能作为依赖？

Python 类是可调用对象，FastAPI 可以分析它的 __init__ 参数并创建实例。

3. yield 依赖适合什么场景？

适合需要请求结束后清理的资源，例如数据库 Session、文件句柄、临时上下文。

⑨ 面试高频真题+解析

1. FastAPI 依赖注入如何提升可测试性？

依赖可以通过 app.dependency_overrides 替换，测试能把真实数据库、外部服务或鉴权逻辑替换成可控对象。

2. 为什么数据库 Session 不建议做成简单全局变量？

全局 Session 生命周期不清、并发不安全、事务边界混乱，也难以在测试中隔离。

3. 依赖注入和中间件有什么区别？

中间件包裹所有请求，适合跨接口逻辑；依赖按路径函数声明解析，适合接口级资源、鉴权、参数和业务前置条件。
