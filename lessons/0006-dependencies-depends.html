Lesson 0006

Depends 依赖注入：把重复参数和公共逻辑交给 FastAPI

本节学习 FastAPI 非常重要的机制：依赖注入。先别被名字吓到，它本质上是“路径函数声明自己需要什么，FastAPI 自动帮它准备好”。

0. 本节学习目标

  • 理解依赖注入要解决的核心问题：重复逻辑和公共资源。
  • 理解 Depends 的执行流程。
  • 看懂类依赖 ItemFilters 和 Pagination。
  • 知道依赖注入未来如何服务数据库连接、鉴权和 AI 模型客户端。

1. 背景问题：查询参数越来越多怎么办

第 3 课里，list_items 直接写了很多查询参数：

def list_items(
    q: str | None = None,
    category: ItemCategory | None = None,
    max_price: float | None = None,
    in_stock: bool | None = None,
) -> list[Item]:
    ...

接口少的时候没问题。但如果多个接口都需要 q、category、分页参数、用户信息、数据库连接，每个函数都重复写一遍，代码会膨胀。

依赖注入就是为了解决这类问题：把公共准备工作放到一个“依赖”里，路径函数只声明自己需要它。

2. Depends 的核心思想

FastAPI 官方文档说，依赖注入表示路径操作函数可以声明自己运行所需的依赖，然后 FastAPI 负责提供这些依赖。

你可以把流程理解成：

请求进入
  |
  v
FastAPI 看到 Depends(...)
  |
  v
先执行依赖函数或依赖类
  |
  v
把依赖返回值注入到路径函数参数里
  |
  v
执行真正的路径函数

所以 Depends 不是装饰器，也不是魔法变量。它是告诉 FastAPI：“这个参数不要直接从请求体拿，而是先运行这个可调用对象来得到”。

3. 本项目的依赖：筛选条件

first_api/dependencies.py 里有一个类依赖：

class ItemFilters:
    def __init__(
        self,
        q: Annotated[str | None, Query(max_length=40)] = None,
        category: ItemCategory | None = None,
        max_price: Annotated[float | None, Query(ge=0)] = None,
        in_stock: bool | None = None,
    ) -> None:
        self.q = q
        self.category = category
        self.max_price = max_price
        self.in_stock = in_stock

类也可以作为依赖，因为 Python 类是可调用对象。FastAPI 会像处理路径函数一样分析 __init__ 的参数，从查询字符串里取值、做校验，然后创建一个 ItemFilters 对象。

这样，筛选参数被收纳到一个对象里，路径函数就不用散着接收四五个参数了。

4. 本项目的依赖：分页参数

class Pagination:
    def __init__(
        self,
        skip: Annotated[int, Query(ge=0)] = 0,
        limit: Annotated[int, Query(ge=1, le=50)] = 10,
    ) -> None:
        self.skip = skip
        self.limit = limit

skip 表示跳过多少条，limit 表示最多返回多少条。它们也是后端列表接口很常见的分页参数。

  • skip=0：默认从第 0 条开始。
  • limit=10：默认最多返回 10 条。
  • Query(ge=0)：不能跳过负数条。
  • Query(ge=1, le=50)：每次至少返回 1 条，最多返回 50 条。

5. 路径函数怎么使用依赖

@router.get("", response_model=list[Item])
def list_items(
    filters: Annotated[ItemFilters, Depends(ItemFilters)],
    pagination: Annotated[Pagination, Depends(Pagination)],
) -> list[Item]:
    results = list(ITEMS.values())
    ...
    return results[pagination.skip : pagination.skip + pagination.limit]

这段签名的意思是：

  • filters 参数由 ItemFilters 这个依赖创建。
  • pagination 参数由 Pagination 这个依赖创建。
  • 路径函数内部使用 filters.q、pagination.limit 这种对象属性。

写成依赖后，未来如果另一个接口也需要同样的筛选或分页逻辑，可以直接复用这些类。

6. 为什么 AI 服务也会大量用 Depends

依赖注入在 AI 服务里非常常见。以后你可能会用它准备：

  • 数据库会话：每个请求拿一个数据库连接，用完释放。
  • 当前用户：从 token 里解析用户身份。
  • 权限检查：判断用户能不能调用某个接口。
  • 模型客户端：创建 OpenAI、向量数据库或本地推理服务客户端。
  • 公共参数：分页、排序、trace id、请求来源等。

提前建立直觉：路径函数应该专注业务逻辑，公共准备工作交给依赖。

7. 实验：分页和筛选一起用

在 /docs 里测试：

GET /items?skip=0&limit=2
GET /items?skip=1&limit=2
GET /items?category=course&skip=0&limit=10
GET /items?limit=0
GET /items?limit=100

前 3 个请求应该成功，后 2 个请求应该返回 422，因为 limit 不满足 ge=1 或 le=50。

8. 课堂练习

请你完成两个小任务：

  1. 用 GET /items?in_stock=true&limit=1 查询库存商品的第一页。
  2. 打开 first_api/dependencies.py，把 limit 的最大值从 50 改成 20，重启服务后测试 limit=30。

完成标准：你能解释为什么修改依赖类会影响使用它的接口。

9. 小测：先回忆再展开

1. Depends 解决的核心问题是什么？

它把公共准备逻辑抽出来，让路径函数声明自己需要什么，由 FastAPI 自动执行并注入。

2. 类为什么也能作为依赖？

因为 Python 类是可调用对象，FastAPI 可以分析它的 __init__ 参数并创建实例。

3. 依赖注入未来和数据库有什么关系？

可以用依赖为每个请求创建数据库会话，并在请求结束后统一清理。

10. 本节总结

Depends 是 FastAPI 的自动装配机制。你声明路径函数需要 ItemFilters 和 Pagination，FastAPI 就负责从请求中解析参数、校验、创建对象并传入函数。掌握它之后，数据库、鉴权、AI 客户端都会更容易理解。

推荐主资源：FastAPI Dependencies、FastAPI Classes as Dependencies。
