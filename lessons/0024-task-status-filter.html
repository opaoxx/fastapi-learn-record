Lesson 0024

任务状态筛选：只看 queued、completed 或 failed

前端任务历史页通常需要“全部、处理中、成功、失败”这些筛选按钮。本节给 GET /tasks 增加 status 查询参数。

0. 本节学习目标

  • 理解状态筛选为什么属于查询参数。
  • 用 TaskStatus 限制合法状态值。
  • 看懂 SQLModel 的 where() 条件查询。
  • 理解筛选和分页如何组合。

1. 筛选不是路径参数

任务列表的资源路径仍然是 /tasks。状态只是“怎么查看这个列表”的条件，所以适合放在查询参数里：

GET /tasks?status=completed
GET /tasks?status=failed
GET /tasks?status=queued

这和前面学过的搜索、分类、分页一样，都是列表视图的查询条件。

2. 用 Literal 限制状态

TaskStatus = Literal["queued", "running", "completed", "failed"]

路由参数写成：

task_status: Annotated[
    TaskStatus | None,
    Query(alias="status"),
] = None

None 表示不筛选，也就是返回所有状态。传了 status 时，只允许四个合法值。比如 /tasks?status=unknown 会自动返回 422。

3. 为什么 Python 变量叫 task_status

代码里已经从 FastAPI 导入了 status 模块，用来写 status.HTTP_202_ACCEPTED。如果参数也叫 status，读代码时容易混乱。

所以我们在 Python 里叫 task_status，但对外 URL 仍然叫 status：

Query(alias="status")

4. service 中拼接 where 条件

statement = select(SummaryTask).order_by(SummaryTask.id.desc())
if task_status is not None:
    statement = statement.where(SummaryTask.status == task_status)
statement = statement.offset(offset).limit(limit)

如果没有传状态，就不加 where；如果传了状态，就只返回这个状态的任务。最后仍然继续分页。

5. 筛选和分页组合

GET /tasks?status=completed&limit=10&offset=0
GET /tasks?status=failed&limit=10&offset=10

执行顺序可以这样理解：

  1. 先按状态筛选出候选任务。
  2. 再按 id desc 排序。
  3. 最后按 offset 和 limit 截取本页。

6. 测试覆盖

completed = client.get("/tasks?status=completed&limit=10", headers=API_KEY_HEADERS)
invalid = client.get("/tasks?status=unknown", headers=API_KEY_HEADERS)

assert completed.status_code == 200
assert all(task["status"] == "completed" for task in completed.json()["items"])
assert invalid.status_code == 422

这个测试同时确认了两件事：合法状态能筛选，非法状态会被 FastAPI 自动拒绝。第 25 课会进一步解释为什么列表数据现在放在 items 字段里。

7. 课堂练习

请你分别请求：

GET /tasks?status=completed
GET /tasks?status=failed
GET /tasks?status=unknown

观察前两个请求返回列表，第三个请求返回 422。然后思考：如果前端有“失败任务”按钮，它应该调用哪个 URL？

8. 小测：先回忆再展开

1. 为什么状态筛选适合做查询参数？

因为它是在查看同一个任务列表时附加的筛选条件。

2. TaskStatus | None 里的 None 表示什么？

表示没有传状态筛选条件，返回所有状态的任务。

3. 非法状态为什么会返回 422？

因为 FastAPI 会根据 Literal 类型自动校验查询参数。

9. 本节总结

本节把任务列表接口升级为“可筛选、可分页”的列表接口。这是前端任务中心、后台管理页、日志系统和 AI 作业历史都需要的基础能力。

推荐主资源：FastAPI Query Parameters、SQLModel Limit and Offset。
