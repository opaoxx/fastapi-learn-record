Lesson 0021

AI 客户端错误：把模型失败变成稳定 API 响应

真实模型会失败：超时、限流、服务不可用、输出异常。本节学习如何把内部异常转换成调用方看得懂的 HTTP 错误。

0. 本节学习目标

  • 理解为什么不能把 Python 原始异常直接暴露给前端。
  • 学习自定义 AIClientError。
  • 用 HTTPException 返回稳定的 503 错误。
  • 理解错误响应模型为什么也是 API 契约的一部分。

1. 为什么 AI 接口更需要错误契约

普通 CRUD 接口常见错误比较固定：资源不存在、参数错误、未授权。AI 接口多了一类问题：后端自己收到了合法请求，但模型服务暂时不能给出结果。

例如：

  • 模型服务超时。
  • 第三方 API 限流。
  • 模型输出格式不符合预期。
  • GPU 或远程推理服务不可用。

这些不是用户请求格式错了，所以不适合返回 422。它们通常更接近 503 Service Unavailable。

2. 自定义异常：AIClientError

class AIClientError(Exception):
    def __init__(
        self,
        message: str = "The AI service is temporarily unavailable.",
        error_code: str = "ai_client_unavailable",
    ) -> None:
        super().__init__(message)
        self.message = message
        self.error_code = error_code

这个异常有两个稳定字段：

  • error_code：给程序判断，例如 demo_ai_failure。
  • message：给人阅读，例如“模型暂时不可用”。

这比直接返回 str(exc) 更稳定，因为前端可以根据 error_code 做提示、重试或日志分类。

3. 错误响应模型

class ErrorDetail(BaseModel):
    error_code: str
    message: str

class ErrorResponse(BaseModel):
    detail: ErrorDetail

FastAPI 的 HTTPException 默认把错误内容放进 detail 字段，所以这里的响应模型也按这个形状设计。

{
  "detail": {
    "error_code": "demo_ai_failure",
    "message": "The demo AI client was asked to simulate a failure."
  }
}

4. 在 /predict 中转换异常

@router.post(
    "/predict",
    response_model=PredictionResponse,
    responses={503: {"model": ErrorResponse}},
)
def predict(...):
    try:
        prediction = ai_client.predict_sentiment(payload.text, payload.mode)
    except AIClientError as exc:
        raise HTTPException(
            status_code=503,
            detail={
                "error_code": exc.error_code,
                "message": exc.message,
            },
        ) from exc

这里有两个动作：内部用 AIClientError 表示模型失败；对外用 HTTPException 表示 HTTP 响应失败。内部异常和外部响应不要混在一起。

5. 为什么是 503

503 Service Unavailable 表示服务当前无法处理请求，通常是临时状态。AI 模型服务超时、临时不可用、限流时，用 503 比 500 更友好。

区别可以这样记：

  • 422：用户请求内容不合法。
  • 401：用户没通过鉴权。
  • 404：资源不存在。
  • 503：后端依赖的服务暂时不可用。

6. 测试错误响应

class FailingAIClient:
    def predict_sentiment(self, text: str, mode: str) -> PredictionResult:
        raise AIClientError(
            message="The test AI client is unavailable.",
            error_code="test_ai_unavailable",
        )

app.dependency_overrides[get_ai_client] = override_failing_ai_client

测试里不用真的让模型挂掉，而是用依赖覆盖注入一个必定失败的客户端。这样错误路径也能被稳定测试。

7. 课堂练习

在 /predict 中发送包含 simulate-ai-failure 的文本：

{
  "text": "please simulate-ai-failure for this request",
  "mode": "fast",
  "temperature": 0.2
}

观察响应状态码和 detail.error_code。

8. 小测：先回忆再展开

1. 为什么不直接把原始异常返回给前端？

因为原始异常不稳定，可能泄露内部实现，也不方便前端按错误类型处理。

2. error_code 和 message 分别给谁用？

error_code 主要给程序判断，message 主要给人阅读。

3. 模型服务临时不可用为什么适合 503？

因为请求本身可能合法，只是后端依赖的服务当前无法完成处理。

9. 本节总结

本节让同步 AI 接口具备了稳定失败契约：内部抛 AIClientError，router 捕获后返回 503 和结构化 detail。
