FastAPI lifespan：应用级 AsyncClient 的创建、复用与关闭

HTTP client 不是普通临时变量。真实服务应复用连接池、集中 timeout/proxy 配置，并在应用关闭时释放资源。本节用 FastAPI lifespan 创建 provider httpx.AsyncClient，存入 app.state，并在 shutdown 阶段 aclose()。

① 本节核心知识框架

Lifespan HTTP Client
├─ startup：create_db_and_tables()
├─ startup：build_ai_client_config()
├─ startup：httpx.AsyncClient(timeout, trust_env=False)
├─ 存储：app.state.provider_http_client
├─ 元信息：provider / timeout_seconds
├─ runtime：请求处理期间复用 client
├─ shutdown：await client.aclose()
└─ 教学探针：GET /provider/http-client

② 核心概念底层原理

HTTPX client 内部维护连接池。跨请求复用 client 可以减少重复连接开销，并集中管理 timeout、代理、headers 等配置。每次请求都创建 client，不仅浪费资源，也容易忘记关闭。

FastAPI lifespan 是应用资源生命周期边界。yield 前运行 startup 逻辑，yield 后运行 shutdown 清理逻辑。它适合管理数据库连接池、HTTP client、模型加载结果、缓存客户端、队列连接等应用级资源。

app.state 是应用级状态容器，适合保存“应用启动一次、跨请求复用、应用关闭时释放”的资源。它不适合保存用户级、请求级或会频繁变化的业务数据。

③ 全套代码逐行精读解析

@asynccontextmanager
async def lifespan(app: FastAPI) -> AsyncGenerator[None, None]:
    create_db_and_tables()
    ai_client_config = build_ai_client_config()
    app.state.provider_http_client = httpx.AsyncClient(
        timeout=ai_client_config.timeout_seconds,
        trust_env=False,
    )
    app.state.provider_http_client_provider = ai_client_config.provider
    app.state.provider_http_client_timeout_seconds = ai_client_config.timeout_seconds
    try:
        yield
    finally:
        await app.state.provider_http_client.aclose()
  • @asynccontextmanager：把函数变成 async 上下文管理器，FastAPI 可用于 lifespan。
  • create_db_and_tables()：启动时初始化本地教学数据库结构。
  • build_ai_client_config()：读取当前 AI provider 配置，保证 HTTP client timeout 与 provider 配置一致。
  • httpx.AsyncClient(...)：创建长期复用的异步 HTTP client。
  • timeout=...：所有 provider 请求共享超时边界。
  • trust_env=False：避免本机代理环境影响教学项目和测试。
  • app.state.provider_http_client：把 client 放到应用级状态中，供后续依赖读取。
  • yield：应用进入运行阶段，可以处理请求。
  • finally：应用退出时保证执行清理。
  • await ...aclose()：关闭连接池和底层资源；漏掉可能造成资源泄漏。
app = FastAPI(
    ...,
    lifespan=lifespan,
)

把 lifespan 注册到 app 后，TestClient context、开发服务器启动/关闭都会触发对应生命周期。若漏传 lifespan，app.state.provider_http_client 不会被创建。

④ 核心机制运行全流程拆解

应用启动
  -> FastAPI 调用 lifespan(app)
  -> 创建 DB 表
  -> 读取 AI client config
  -> 创建 AsyncClient(timeout, trust_env=False)
  -> 保存到 app.state
  -> yield 后开始处理请求

运行期间
  -> endpoint / dependency 从 app.state 读取 client
  -> provider 调用复用连接池

应用关闭
  -> lifespan 继续执行 yield 后代码
  -> await provider_http_client.aclose()
  -> 释放连接资源

⑤ 重难点、易混淆点对比辨析

概念正确理解常见误区
lifespan应用级资源生命周期普通业务函数
app.state应用级资源存储存请求级数据
AsyncClient可复用连接池资源每次请求随手创建
aclose()释放连接资源可省略
trust_env是否读取环境代理与请求业务无关的小细节

⑥ 开发常见报错+坑点+解决方案

  • AttributeError: State object has no attribute provider_http_client：lifespan 未执行或未注册。解决：使用 FastAPI(lifespan=lifespan)，测试中用 with TestClient(app)。
  • 连接泄漏：创建 client 未关闭。解决：shutdown 阶段 await aclose()。
  • 代理环境导致 client 创建失败：HTTPX 默认读取环境代理。解决：教学环境设置 trust_env=False。
  • 请求间数据串扰：把请求级状态放到 app.state。解决：请求数据放函数局部变量、依赖或数据库。
  • 测试没有触发生命周期：直接调用 app 或不用 context。解决：使用 with TestClient(app)。

⑧ 课后记忆习题+标准答案

1. lifespan 中 yield 前后分别做什么？

yield 前执行应用启动逻辑，yield 后执行应用关闭清理逻辑。

2. 为什么 AsyncClient 适合跨请求复用？

它维护连接池，复用可以减少重复连接开销，并集中配置 timeout/proxy 等策略。

3. app.state 适合保存什么？

适合保存应用级、跨请求复用、生命周期由应用管理的资源。

4. 为什么教学项目设置 trust_env=False？

为了避免本机代理环境变量影响 HTTPX client 创建和测试稳定性。

⑨ 面试高频真题+解析

题 1：FastAPI 中如何管理长期 HTTP client？

在 lifespan startup 创建 client，存入 app.state 或容器；请求期间通过依赖读取；shutdown 时统一 aclose，避免每个请求创建和泄漏资源。

题 2：为什么不建议每个请求新建 AsyncClient？

无法复用连接池，增加连接开销，配置散落，且容易忘记关闭导致资源泄漏。

题 3：多 worker 部署下 lifespan 资源是什么范围？

通常每个进程/worker 各自运行 lifespan 并拥有自己的 client/连接池，不是所有 worker 共享一个 Python 对象。
