Lesson 0015

文件上传 UploadFile：让 API 接收文本文件

前面接口主要接收 JSON。真实 AI 服务常常需要接收文件：文档、图片、音频、表格。本节先从最简单的 UTF-8 文本文件开始。

0. 本节学习目标

  • 理解文件上传为什么使用 multipart/form-data，而不是普通 JSON。
  • 理解 UploadFile 和 File() 的作用。
  • 看懂 POST /files/text 如何校验文件名、大小和编码。
  • 理解为什么响应模型只返回文件元数据和预览，不返回完整内容。

1. 为什么文件上传不是 JSON

JSON 很适合传结构化文本，例如商品名称、价格、推理参数。但文件可能很大，也可能是二进制数据，直接塞进 JSON 不自然。

浏览器和 HTTP 客户端通常用 multipart/form-data 上传文件。FastAPI 官方文档也说明，接收上传文件时使用 File、bytes 或 UploadFile，文件会作为表单数据发送。

2. UploadFile 是什么

async def upload_text_file(
    file: Annotated[UploadFile, File(description="A UTF-8 .txt file.")],
    session: Annotated[Session, Depends(get_session)],
) -> UploadedTextFile:
    ...

UploadFile 表示上传来的文件对象。它包含：

  • filename：客户端上传的文件名。
  • content_type：客户端声明的媒体类型。
  • read()：异步读取文件内容。

File() 告诉 FastAPI：这个参数来自文件表单字段，而不是 JSON 请求体。

3. 文件接口为什么也要 API Key

上传文件会占用存储、数据库和后续 AI 处理成本，所以 /files 路由也被 X-API-Key 保护：

router = APIRouter(
    prefix="/files",
    tags=["files"],
    dependencies=[Depends(require_api_key)],
)

这和第 12 课的写接口保护是同一个原则：会消耗资源或改变系统状态的接口，需要先做基本鉴权。

4. 校验：文件名、大小、编码

if not file.filename or not file.filename.lower().endswith(".txt"):
    raise HTTPException(status_code=422, detail="Only .txt files are accepted.")

raw_content = await file.read()
if len(raw_content) > MAX_TEXT_FILE_BYTES:
    raise HTTPException(status_code=413, detail="Text file is too large.")

content = raw_content.decode("utf-8")

这三个校验很重要：

  • 只收 .txt，降低学习阶段的复杂度。
  • 限制大小，避免一个请求塞爆内存或数据库。
  • 要求 UTF-8，保证后续摘要任务能按文本处理。

5. 数据库存什么

class UploadedTextFile(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True)
    filename: str
    content_type: str
    size_bytes: int
    content: str
    preview: str

教学阶段我们把文本内容直接存在 SQLite。真实项目里，较大的文件通常会放到对象存储，例如 S3、OSS、MinIO，数据库只保存文件 URL、大小、类型和归属信息。

6. 为什么响应不返回完整 content

class UploadedTextFileRead(SQLModel):
    id: int
    filename: str
    content_type: str
    size_bytes: int
    preview: str

响应模型不包含完整 content，只返回 preview。这是为了避免把用户上传的完整内容随便暴露出去，也避免响应过大。

7. 实验：上传文本文件

在 /docs 里找到 POST /files/text。先点击 Authorize 填入 dev-secret-key，然后上传一个 .txt 文件。

你应该得到 201，响应里包含 id、filename、size_bytes 和 preview。

8. 课堂练习

请你测试：

  1. 上传 notes.txt，观察 201。
  2. 上传 notes.md，观察 422。
  3. 不带 API Key 上传，观察 401。

9. 小测：先回忆再展开

1. 文件上传为什么常用 multipart/form-data？

因为文件不是普通 JSON 字段，表单文件格式更适合传输文件内容和文件元数据。

2. UploadFile 比普通字符串多了什么？

它带有文件名、内容类型和读取文件内容的方法。

3. 为什么响应只返回 preview？

为了避免暴露完整上传内容，并控制响应体大小。

10. 本节总结

UploadFile 是 FastAPI 接收文件上传的核心工具。文件上传接口要考虑鉴权、文件类型、大小限制、编码和响应暴露范围。下一节会把上传文件接到后台摘要任务中。
