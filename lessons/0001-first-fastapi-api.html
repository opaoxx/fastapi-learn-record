Lesson 0001

跑起来你的第一个 FastAPI API

目标很小：理解一个接口从代码到浏览器访问的完整闭环。这个闭环以后会变成普通后端接口、AI 推理接口、求职项目接口。

你今天只需要掌握一件事

FastAPI 的核心动作是：创建一个 app，把一个 URL 和一个 Python 函数绑定起来，然后让服务器接收 HTTP 请求。

官方说法对照：FastAPI 官方教程把这叫做 path operation。路径是 /health 这样的 URL 片段，operation 是 GET、POST 这样的 HTTP 方法，函数就是请求到来后被调用的 Python 函数。

先看代码

工作区里已经有一个小示例：first_api/main.py。

from fastapi import FastAPI
from pydantic import BaseModel, Field

app = FastAPI(title="First FastAPI API")

@app.get("/")
def read_root():
    return {"message": "Hello FastAPI"}

@app.get("/health")
def health_check():
    return {"status": "ok", "service": "first-api"}

这两条接口都是普通后端最常见的 GET 接口。根接口确认服务能访问，健康检查接口常用于部署、监控和面试项目。

跑起来

在 PowerShell 里进入 F:\teaching_me，执行：

python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install "fastapi[standard]"
fastapi dev first_api/main.py

成功后，浏览器打开：

  • http://127.0.0.1:8000/
  • http://127.0.0.1:8000/health
  • http://127.0.0.1:8000/docs

/docs 是 FastAPI 自动生成的交互式 API 文档。以后做求职项目时，它会帮你展示接口能力。

为什么这能通向 AI 服务

AI 服务接口本质上也是 API：用户发一段文本、图片地址或参数，后端调用模型，再把结果返回。示例里已经放了一个假的 /predict 接口，用来提前建立这个感觉。

@app.post("/predict")
def predict(payload: PredictionRequest) -> PredictionResponse:
    ...

先不用管模型怎么接。你今天只要记住：AI 推理函数外面包一层 FastAPI，就能被前端、脚本或其他服务调用。

练习：改一个接口

打开 first_api/main.py，把根接口返回的 Hello FastAPI 改成你自己的句子，然后重新刷新浏览器。

完成标准：你能在浏览器看到新的 JSON 返回。

小测：遮住上面内容再答

1. @app.get("/health") 这行在做什么？

它把 GET 请求的 /health 路径绑定到下面那个 Python 函数。

2. 为什么返回字典时浏览器看到的是 JSON？

FastAPI 会把可序列化的 Python 返回值转换成 HTTP 响应里的 JSON。

3. /docs 对求职项目有什么用？

它自动展示接口、参数和返回结构，让别人能直接试用你的 API。
