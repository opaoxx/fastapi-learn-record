FastAPI async 基础：事件循环、await 与阻塞边界

async 不是 CPU 加速器，而是等待型 I/O 的并发工具。当外部 AI provider、HTTP 服务或异步数据库在等待网络响应时，await 让当前协程交还执行权，事件循环可以处理其他请求。本节用 GET /async/wait 建立正确心智模型。

① 本节核心知识框架

FastAPI async 心智模型
├─ async def：定义协程函数
├─ await：等待可等待对象并让出执行权
├─ event loop：调度协程任务
├─ 适合 async：网络、文件、外部 HTTP、async DB 等 I/O 等待
├─ 不适合只靠 async：CPU 密集计算、同步阻塞库
├─ 教学端点：GET /async/wait?delay=0.05
├─ 非阻塞等待：await asyncio.sleep(delay)
└─ 参数保护：Query(ge=0, le=1)

② 核心概念底层原理

并发是多个任务交替推进，并行是多个任务在多个 CPU 核上真正同时执行。async 主要解决并发等待：一个请求在等外部 provider 时，线程不必傻等，可以让事件循环调度其他协程。

async def 创建协程函数，调用它得到协程对象；只有被 await、任务调度或框架执行时，协程才真正推进。await 的关键不是“等待”，而是“等待期间让出执行权”。

阻塞调用会破坏 async。time.sleep()、同步 HTTP 请求、同步数据库驱动都会占住执行线程。把外层函数改成 async def，不会让同步库自动非阻塞。

③ 全套代码逐行精读解析

class AsyncWaitResponse(BaseModel):
    status: Literal["ok"]
    delay_seconds: float
    message: str
  • BaseModel：即使是教学接口，也用响应模型固定 JSON 形状。
  • Literal["ok"]：状态字段收窄为唯一成功值，OpenAPI 可见。
  • delay_seconds：回显等待参数，便于测试断言。
  • message：说明该端点是等待演示，不是 CPU 工作。
@router.get(
    "/async/wait",
    response_model=AsyncWaitResponse,
    summary="Demonstrate non-blocking async waiting",
)
async def async_wait(
    delay: Annotated[
        float,
        Query(ge=0, le=1, description="Seconds to await. Kept small so the learning endpoint stays safe."),
    ] = 0.05,
) -> AsyncWaitResponse:
    await asyncio.sleep(delay)
    return AsyncWaitResponse(
        status="ok",
        delay_seconds=delay,
        message="The endpoint awaited without doing CPU work.",
    )
  • @router.get 注册公开教学端点。
  • response_model 让响应进入 OpenAPI 契约。
  • async def 允许函数体内使用 await。
  • Query(ge=0, le=1) 防止用户请求超长等待。
  • await asyncio.sleep(delay) 模拟非阻塞 I/O 等待；等待时事件循环可处理别的任务。
  • return AsyncWaitResponse(...) 返回显式模型，避免随手 dict 字段漂移。

④ 核心机制运行全流程拆解

GET /async/wait?delay=0.05
  -> FastAPI 解析 query 字符串为 float
  -> Query 校验 0 <= delay <= 1
  -> 调用 async_wait 协程
  -> 执行到 await asyncio.sleep(delay)
       -> 当前协程暂停
       -> 事件循环可调度其他任务
  -> sleep 完成后协程恢复
  -> 返回 AsyncWaitResponse
  -> FastAPI 序列化 JSON

⑤ 重难点、易混淆点对比辨析

概念正确理解错误理解
async提升等待型并发效率让 CPU 计算更快
await等待并让出执行权普通阻塞等待
asyncio.sleep非阻塞等待和 time.sleep 一样
同步 DB仍然阻塞外层 async 后自动异步
CPU 密集任务线程/进程/队列/推理服务简单加 async

⑥ 开发常见报错+坑点+解决方案

  • coroutine was never awaited：调用协程未 await。解决：在 async 上下文使用 await 或创建任务。
  • async endpoint 仍然卡住：内部调用同步阻塞库。解决：改用 async client，或把阻塞工作移到线程/进程/队列。
  • delay=2 返回 422：这是预期保护。解决：教学端点只允许 0 到 1 秒。
  • 以为 async 等于多线程：async 是协作式调度，不是自动并行。
  • 同步 SQLModel 直接改 async：数据库调用并未异步。解决：需要 async engine/session/driver 成套迁移。

⑧ 课后记忆习题+标准答案

1. async 的核心价值是什么？

在等待 I/O 时让出执行权，提高并发处理效率，而不是加速 CPU 计算。

2. await asyncio.sleep 和 time.sleep 区别是什么？

前者非阻塞并让事件循环调度其他任务；后者阻塞当前线程。

3. 为什么同步数据库不会因外层 async 自动变异步？

阻塞行为由内部库决定；同步 driver 仍占用线程，外层函数签名无法改变它。

4. 什么时候普通 def 反而更清楚？

当函数内部全是同步库且没有可 await I/O 时，普通 def 更符合实际执行模型。

⑨ 面试高频真题+解析

题 1：FastAPI 中 def 和 async def 如何选择？

内部使用 async I/O 客户端时选 async def；内部全是同步阻塞库时普通 def 更清晰。不要在 async endpoint 中执行长时间阻塞调用。

题 2：async 能解决 CPU 密集任务吗？

不能。CPU 密集任务需要并行计算、进程池、任务队列或专门推理服务，async 只改善等待型并发。

题 3：事件循环一次能运行多少 Python task？

单个事件循环同一时刻运行一个 task；task 在 await 处暂停后，事件循环调度其他 task。它是协作式并发，不是自动多核并行。
