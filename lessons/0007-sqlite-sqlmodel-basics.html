Lesson 0007

SQLite 和 SQLModel：把数据从内存搬进数据库

前面的商品数据存在 Python 字典里，服务重启就会回到初始状态。本节学习数据库的最小闭环：SQLite 文件、SQLModel 表模型、engine、建表和种子数据。

0. 本节学习目标

  • 理解为什么后端项目需要数据库，而不是只用内存变量。
  • 理解 SQLite 是一个适合入门的本地文件数据库。
  • 理解 SQLModel 如何把 Python 类映射成数据库表。
  • 看懂 database.py 里创建 engine、建表、初始化数据的流程。

1. 为什么需要数据库

内存字典很适合教学，但它有一个致命问题：数据只活在当前 Python 进程里。你创建一个商品，重启服务后它就消失了。

数据库解决的是“持久化”问题：把数据保存到进程外部。服务重启、代码重载、电脑关机后，只要数据库文件还在，数据就还在。

普通后端视角：用户、商品、订单、评论都应该放在数据库里。
AI 服务视角：推理任务、上传文件、模型调用记录、向量索引元数据也需要持久化。

2. SQLite 是什么

SQLite 是一个轻量级关系数据库。它不需要单独启动数据库服务器，数据通常保存在一个本地 .db 文件里。FastAPI 官方 SQL 数据库教程也选择 SQLite 作为入门示例，因为它可以直接复制运行。

本项目会在 first_api/first_api.db 创建数据库文件。这个文件是运行产物，所以已经加入 .gitignore，通常不提交到 GitHub。

3. SQLModel 是什么

SQLModel 是 FastAPI 作者推出的数据库库，建立在 SQLAlchemy 和 Pydantic 之上。你可以先这样理解：

  • Pydantic 负责数据校验和 API schema。
  • SQLAlchemy 负责和 SQL 数据库交互。
  • SQLModel 把两者结合起来，让一个 Python 类既像 API 模型，也能映射数据库表。

例如现在的表模型：

class Item(ItemBase, table=True):
    id: int | None = Field(default=None, primary_key=True)

table=True 是关键：它告诉 SQLModel，这个类不仅是数据模型，还对应数据库里的一张表。

4. 三类 Item 模型

现在 schemas.py 里不再只有一个 Item，而是拆成几层：

class ItemBase(SQLModel):
    name: str
    category: str = Field(index=True)
    price: float = Field(ge=0)
    in_stock: bool

class Item(ItemBase, table=True):
    id: int | None = Field(default=None, primary_key=True)

class ItemCreate(ItemBase):
    category: ItemCategory = "course"

class ItemRead(ItemBase):
    id: int

这样拆是为了区分不同场景：

  • ItemBase：公共字段。
  • Item：数据库表模型，有主键 id。
  • ItemCreate：创建商品时客户端能提交的字段。
  • ItemRead：接口返回给客户端的字段。

注意：数据库表里的 category 用普通字符串保存，因为数据库列需要明确的 SQL 类型；创建请求里仍然用 Literal["book", "course", "tool"] 做输入校验。

5. database.py 逐段解释

DATABASE_URL = f"sqlite:///{Path(__file__).with_name('first_api.db')}"
engine = create_engine(
    DATABASE_URL,
    connect_args={"check_same_thread": False},
)

DATABASE_URL 告诉 SQLModel 连接哪个数据库。sqlite:///... 表示使用 SQLite 文件。engine 可以理解成“数据库连接工厂”，后面所有 session 都通过它和数据库交流。

def create_db_and_tables() -> None:
    SQLModel.metadata.create_all(engine)
    with Session(engine) as session:
        seed_items(session)

create_all 会根据所有 table=True 的 SQLModel 模型创建数据库表。seed_items 会在表为空时放入三条初始商品数据。

6. 应用启动时初始化数据库

main.py 现在使用 lifespan：

@asynccontextmanager
async def lifespan(app: FastAPI) -> AsyncGenerator[None, None]:
    create_db_and_tables()
    yield

app = FastAPI(..., lifespan=lifespan)

这表示应用启动时先创建表和初始化数据，然后再开始接收请求。后面的 yield 表示应用正常运行阶段。

7. 实验：观察数据库文件

运行服务：

.\.venv\Scripts\Activate.ps1
fastapi dev first_api/main.py

启动后，查看 first_api/first_api.db 是否出现。然后打开 /docs 测试：

GET /items
GET /items/2

如果能返回商品，说明 FastAPI 已经从数据库读取数据，而不是从内存字典读取。

9. 小测：先回忆再展开

1. table=True 表示什么？

它表示这个 SQLModel 类会映射成数据库表。

2. engine 可以先理解成什么？

可以理解成数据库连接工厂，负责让 session 和数据库建立联系。

3. SQLite 为什么适合入门？

因为它不需要单独数据库服务器，数据保存在一个本地文件里，运行和学习成本低。

10. 本节总结

数据库让数据离开内存，获得持久化能力。SQLite 是适合入门的本地数据库，SQLModel 让 Python 类型、Pydantic 校验和 SQL 表结构结合起来。下一节会把接口真正改成数据库 CRUD。
