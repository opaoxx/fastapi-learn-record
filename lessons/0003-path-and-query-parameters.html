Lesson 0003

路径参数和查询参数：让 URL 会表达意图

本节学习 FastAPI 最常用的两类 URL 参数：路径参数负责定位一个资源，查询参数负责筛选一批资源。理解它们，你就能读懂大多数 REST 风格接口。

0. 本节学习目标

  • 理解路径参数和查询参数在 URL 中的位置与职责。
  • 能写出 GET /items/{item_id} 这种“按 ID 查单个资源”的接口。
  • 能写出 GET /items?category=book&max_price=50 这种“按条件筛选列表”的接口。
  • 理解 FastAPI 如何根据函数参数名、类型注解和默认值判断参数来源。

1. 背景知识：URL 不是随便写的字符串

后端接口的 URL 应该表达“你要操作什么资源”。例如：

GET /items/2
GET /items?category=book&max_price=50&in_stock=true

第一行里的 2 是路径的一部分，表示要找 ID 为 2 的商品。第二行里的 category、max_price、in_stock 在问号后面，表示对商品列表进行筛选。

一句话区分：路径参数用于“找哪一个”，查询参数用于“怎么筛选”。

2. 路径参数：定位一个具体资源

FastAPI 用花括号声明路径参数：

@app.get("/items/{item_id}", response_model=Item)
def read_item(
    item_id: Annotated[int, Path(ge=1, description="The numeric ID of the item.")]
) -> Item:
    return get_item_or_404(item_id)

这里有三层含义：

  1. "/items/{item_id}" 声明 URL 里有一个动态片段。
  2. item_id: int 要求 FastAPI 把 URL 字符串转换成整数。
  3. Path(ge=1) 给路径参数增加规则：ID 必须大于等于 1。

所以访问 /items/2 会得到 ID 为 2 的商品；访问 /items/abc 会失败，因为 abc 不能转换成整数；访问 /items/0 也会失败，因为不满足 ge=1。

3. 查询参数：筛选一批资源

查询参数在 URL 的问号后面，用 & 连接多个条件：

GET /items?q=python&category=book&max_price=50&in_stock=true

对应代码是：

@app.get("/items", response_model=list[Item])
def list_items(
    q: Annotated[str | None, Query(max_length=40)] = None,
    category: ItemCategory | None = None,
    max_price: Annotated[float | None, Query(ge=0)] = None,
    in_stock: bool | None = None,
) -> list[Item]:
    ...

这些参数没有出现在路径模板 "/items" 里，所以 FastAPI 会把它们当成查询参数。默认值是 None，表示这个筛选条件可以不传。

4. FastAPI 的判断规则

先记住这个入门规则，后面会越来越自然：

  • 函数参数名出现在路径里，例如 {item_id}，它就是路径参数。
  • 函数参数是简单类型，并且没有出现在路径里，它通常就是查询参数。
  • 函数参数是 Pydantic 模型，并且用于 POST 这类请求，它通常来自请求体。

因此 read_item(item_id: int) 从路径拿 ID；list_items(category=...) 从问号后面拿筛选条件；上一课的 predict(payload: PredictionRequest) 从 JSON 请求体拿数据。

5. 实验一：按 ID 查询

打开 http://127.0.0.1:8000/docs，测试：

GET /items/1
GET /items/2
GET /items/0
GET /items/abc

前两个应该成功，后两个应该返回 422。这里的 422 来自参数校验：URL 能被路由匹配，但参数类型或范围不符合声明。

6. 实验二：筛选列表

测试这些地址：

GET /items
GET /items?category=book
GET /items?max_price=20
GET /items?in_stock=false
GET /items?q=python

你会看到返回列表逐渐变少，因为每个查询参数都在增加一个筛选条件。这个模式以后会出现在搜索页、商品页、日志页、模型任务列表等很多后端接口里。

7. 课堂练习

请你在 /docs 里找出下面请求各自返回几个商品：

GET /items?category=course
GET /items?category=tool&in_stock=false
GET /items?max_price=0
GET /items?q=api

完成标准：你能说出每个查询参数具体过滤了哪个字段。

8. 小测：先回忆再展开

1. /items/{item_id} 中的 {item_id} 是什么？

它是路径参数，用来从 URL 路径中捕获一个动态值。

2. /items?category=book 中的 category 是什么？

它是查询参数，用来筛选或修饰这次列表查询。

3. 查询参数为什么常常有默认值？

因为筛选条件通常是可选的，不传时接口应该返回更宽泛的结果。

9. 本节总结

路径参数定位资源，查询参数筛选资源。FastAPI 通过路径模板、函数参数名、类型注解和默认值来决定从哪里取数据。掌握这一节，你已经能写出常见的“详情接口”和“列表筛选接口”。
