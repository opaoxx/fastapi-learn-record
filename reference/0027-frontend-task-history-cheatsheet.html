Reference 0027

前端任务历史速查

UI 状态是 API 契约的投影：响应信封进来，任务列表和页面元信息出去。

请求任务历史

const response = await fetch("/tasks?limit=5&offset=0", {
  headers: {"X-API-Key": apiKey},
});
const page = await parseJsonResponse(response);

响应信封字段

  • items：当前页任务数组。
  • count：符合当前条件的任务总数。
  • limit：本次最多取多少条。
  • offset：本次从第几条开始取。

fetch 错误边界

async function parseJsonResponse(response) {
  const body = await response.json().catch(() => ({}));
  if (!response.ok) {
    throw new Error(`${response.status} ${response.statusText}: ${JSON.stringify(body)}`);
  }
  return body;
}

渲染列表骨架

taskList.replaceChildren();
historyMeta.textContent =
  `显示 ${page.items.length} 条，共 ${page.count} 条`;

for (const task of page.items) {
  const row = document.createElement("article");
  row.textContent = `任务 #${task.id}: ${task.status}`;
  taskList.append(row);
}

常见错误

  • 把 page 当数组遍历；应该遍历 page.items。
  • 忘记 X-API-Key；受保护路由会返回 401。
  • 不检查 response.ok；错误响应会被误当成功响应。
  • 刷新前不清空旧 DOM；列表会重复追加。
来源：MDN Fetch API、 MDN Response.json()、 MDN createElement()、 FastAPI Response Model。
