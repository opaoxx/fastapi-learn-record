Reference 0035

app.state 依赖桥速查

lifespan 负责资源生命周期，app.state 保存应用级资源，Depends 把资源交给路由使用。

核心心智模型

lifespan owns the resource
app.state stores the resource
dependency exposes the resource
endpoint uses the resource

标准写法

def get_provider_http_client(request: Request) -> httpx.AsyncClient:
    return request.app.state.provider_http_client


def endpoint(
    provider_http_client: Annotated[
        httpx.AsyncClient,
        Depends(get_provider_http_client),
    ],
):
    ...

职责分工

  • lifespan：创建和关闭长期资源。
  • app.state：保存跨请求复用的应用级资源。
  • Depends：把资源作为参数提供给 endpoint。
  • endpoint：使用资源处理 HTTP 请求，不负责资源生命周期。

测试覆盖写法

replacement_client = httpx.AsyncClient(timeout=0.1, trust_env=False)
app.dependency_overrides[get_provider_http_client] = lambda: replacement_client

try:
    with TestClient(app) as client:
        response = client.get("/provider/http-client")
finally:
    app.dependency_overrides.clear()

常见坑

  • 把 request.app.state.xxx 散落在很多路由里。
  • 在依赖函数里每次请求都新建长期 client。
  • 测试后忘记清理 app.dependency_overrides。
  • 把用户级、请求级数据保存到 app.state。

面试短答

lifespan 管理长期资源生命周期，app.state 保存应用级资源，依赖函数把资源暴露给路由。这样路由不依赖资源存储细节，测试也能用 dependency_overrides 替换资源。

来源：FastAPI Dependencies、 FastAPI Testing Dependencies、 FastAPI Lifespan Events、 Starlette Applications。
