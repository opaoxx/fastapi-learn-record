Reference 0077

GitHub Actions CI Workflow 速查

把本地维护命令变成 GitHub PR 质量门禁。

核心文件

.github/workflows/ci.yml

触发器

push: main
pull_request: main

关键步骤

actions/checkout@v4
actions/setup-python@v5
python -m pip install -r requirements.txt
python -m pytest tests/test_provider_http.py -q
python -m pytest tests/test_course_docs_contract.py -q
python -m pytest -q
git diff --check

设计要点

  • CI 是编排层，pytest 是检查工具。
  • runner 是独立临时环境，不能依赖本地虚拟环境。
  • 文档契约测试必须进入 CI，因为课程文档是交付物。
  • 生产环境通常还会增加 lint、type check、security scan、build 和 required checks。
