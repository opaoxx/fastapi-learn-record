Reference 0061

Provider Metrics Documentation Index 速查

文档入口负责把端点、契约、边界、课程和测试连成一张可维护地图。

一句话模型

关键文档不要只写给人看，也要用测试保护它没有漏掉项目契约。

本课新增文件

  • reference/provider-metrics-index.html：provider metrics 长期文档入口。
  • tests/test_course_docs_contract.py：保护文档入口内容和链接。
  • lessons/0061-provider-metrics-documentation-index.html：解释 docs-as-contract 的设计。

文档入口必须包含

类别 必须出现的内容
Endpoint /provider/metrics、/provider/metrics/prometheus
Metric provider_prediction_total
Rename get_provider_prediction_metric_contract
Boundary not a production multi-instance metrics pipeline
Tests test_provider_http.py、test_openapi_contract.py、test_course_docs_contract.py

测试命令

.\.venv\Scripts\python.exe -m pytest tests\test_course_docs_contract.py -q

测试形状

parser = parse_html(Path("reference/provider-metrics-index.html"))

assert "/provider/metrics" in parser.text
assert "provider_prediction_total" in parser.text
assert "../tests/test_provider_http.py" in parser.links

常见错误

  • 把入口写成重复讲义，而不是检索地图。
  • 只链接课程，不链接测试。
  • 漏掉生产边界，误导读者使用教学 counter。
  • 用测试锁死全部文案，造成维护噪音。
来源：Python html.parser、 pytest assertions、 Prometheus instrumentation practices。
