Reference 0029

OpenAPI 契约速查

FastAPI 从代码生成 /openapi.json，/docs 再把它渲染成交互式文档。

核心心智模型

FastAPI code
  -> routes + type hints + models + params + security
  -> /openapi.json
  -> /docs

常见来源

代码 契约含义
FastAPI(title=..., summary=...) API 级元信息
tags=["tasks"] 接口分组
summary / description 路径操作说明
response_model=Model 成功响应 schema
Query(ge=1, le=100) 查询参数校验和文档
APIKeyHeader header API key 安全方案

任务列表契约

GET /tasks?status=completed&limit=5&offset=0
Header: X-API-Key

200 response:
{
  "items": [],
  "count": 0,
  "limit": 5,
  "offset": 0
}

契约测试模板

schema = client.get("/openapi.json").json()
operation = schema["paths"]["/tasks"]["get"]

assert operation["responses"]["200"]["content"]["application/json"]["schema"] == {
    "$ref": "#/components/schemas/SummaryTaskListResponse"
}
assert operation["security"] == [{"ApiKeyAuth": []}]

常见坑

  • 只看 /docs，不理解 /openapi.json 才是机器可读契约。
  • 忘记 response_model，导致响应结构不够清晰。
  • 只写 int，不写 Query 边界。
  • 接口契约变化了，但没有对应测试提醒。
来源：FastAPI Metadata、 FastAPI Path Operation Configuration、 FastAPI Response Model、 FastAPI Security Tools。
