Reference 0064

Provider Metrics Runbook Findings 速查

把 metrics samples 翻译成 finding 和 next_action，但不替代生产告警。

一句话模型

metrics 是事实，finding 是解释，next_action 是下一步排障动作。

核心函数

build_provider_metrics_runbook_findings(samples)

分支映射

outcomeseverity含义下一步
successbaseline成功请求存在。对比预期请求量。
retry_scheduledinvestigate发生过可重试失败并安排重试。查 failure labels、retry delay log、exhausted 样本。
retry_exhaustedaction_required重试预算耗尽。查 max_attempts、provider 可用性、最后错误。
fail_fastcontract_check非重试失败或无效响应。先查请求/响应契约。
unknownunknownrunbook 没有解释分支。先补 runbook 分支。

不要混淆

  • severity 是教学判断级别，不是生产 P1/P2。
  • finding 不应写进 metric label。
  • next_action 是排查建议，不是自动修复指令。
  • 未知 outcome 不能静默忽略。
