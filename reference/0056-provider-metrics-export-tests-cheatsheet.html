Reference 0056

Provider Metrics Export Tests 速查

metrics text export 是机器读取的契约，测试要覆盖空状态、失败状态、转义规则和 HTTP 响应边界。

核心心智模型

不要把 text export 当成普通字符串展示。HELP、TYPE、metric name、label 顺序、label 转义、count 和 Content-Type 都是契约。

测试分层

层级 测试对象 保护重点
helper render_provider_metrics_prometheus_text() 文本格式、空 snapshot、排序、转义
endpoint GET /provider/metrics/prometheus 路由、状态码、Content-Type、真实请求流程

空 snapshot

# HELP provider_prediction_total Provider prediction outcomes recorded by the teaching metrics counter.
# TYPE provider_prediction_total counter

没有样本时仍然保留 HELP 和 TYPE。空状态不是空字符串。

失败 outcome

provider_prediction_total{error_code="ai_provider_http_error",failure_category="retryable_http_status",operation="prediction",outcome="retry_exhausted",status_code="503"} 1
provider_prediction_total{error_code="ai_provider_http_error",failure_category="retryable_http_status",operation="prediction",outcome="retry_scheduled",status_code="503"} 1

转义规则

字符 输出
反斜杠 \\
换行 \n
双引号 \"

endpoint 断言模板

response = client.get("/provider/metrics/prometheus")

assert response.status_code == 200
assert response.headers["content-type"].startswith("text/plain")
assert response.text == expected_text

常见错误

  • 只断言 status code，不断言响应正文。
  • 把空 metrics 导出写成空字符串。
  • 忘记测试失败 outcome。
  • 忘记 label value 转义。
  • endpoint 测试没有清空全局 counter。

运行命令

.\.venv\Scripts\python.exe -m pytest tests\test_provider_http.py tests\test_ai_client_dependency.py -q
