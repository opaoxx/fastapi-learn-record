Reference 0023

任务分页速查

limit 是窗口大小，offset 是窗口起点。分页前必须先有稳定排序。

请求模式

GET /tasks?limit=20&offset=0
GET /tasks?limit=20&offset=20
GET /tasks?limit=20&offset=40

参数校验

offset: Annotated[int, Query(ge=0)] = 0
limit: Annotated[int, Query(ge=1, le=100)] = 20

数据库查询

statement = select(SummaryTask).order_by(SummaryTask.id.desc())
statement = statement.offset(offset).limit(limit)
items = list(session.exec(statement))

判断规则

  • offset=0 是第一页。
  • 下一页通常是 offset + limit。
  • 上一页通常是 max(offset - limit, 0)。
  • 没有稳定 order_by，分页可能重复或漏项。

常见 422

  • limit=0：页大小不能小于 1。
  • limit=101：超过服务端上限。
  • offset=-1：偏移量不能为负。
来源：FastAPI Query Parameters、 FastAPI Query Validations、 SQLModel Limit and Offset。
