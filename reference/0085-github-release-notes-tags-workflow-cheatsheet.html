Reference 0085

GitHub Release Notes and Tags Workflow 速查

用 annotated tag 固定课程批次，用 GitHub Release Notes 记录变更范围、验证证据、已知警告和回滚路径。

核心文件

.github/RELEASE_PROCESS.md

Tag 命名

course-vYYYY.MM.DD-N

example:
course-v2026.07.05-1

Release Notes 结构

Changed Lessons
Changed References
Code and Workflow Changes
Verification
Known Warnings
Upgrade or Reading Notes
Rollback

发布前检查

.\.venv\Scripts\python.exe -m pytest tests\test_provider_http.py -q
.\.venv\Scripts\python.exe -m pytest tests\test_course_docs_contract.py -q
.\.venv\Scripts\python.exe -m pytest -q
git diff --check

关键原则

Use annotated tags.
Do not silently move published tags.
GitHub Releases are remote state.
Release notes must include verification evidence.
