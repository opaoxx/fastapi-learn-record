Reference 0037

Provider Prediction Adapter 速查

把本地预测输入转换成 provider JSON，再把 provider JSON 校验并转换成内部 PredictionResult。

核心链路

text + mode
  -> ProviderPredictionPayload
  -> await client.post(url, json=...)
  -> response.raise_for_status()
  -> ProviderPredictionBody.model_validate(response.json())
  -> PredictionResult

请求模型

class ProviderPredictionPayload(BaseModel):
    text: str
    mode: str

响应校验模型

class ProviderPredictionBody(BaseModel):
    label: PredictionLabel
    score: float = Field(ge=0, le=1)

成功返回

return PredictionResult(
    label=body.label,
    score=round(body.score, 2),
    source=f"{config.provider}-provider",
)

错误映射

  • TimeoutException -> ai_provider_timeout
  • HTTPStatusError -> ai_provider_http_error
  • RequestError -> ai_provider_request_error
  • ValueError / ValidationError -> ai_provider_invalid_response

测试重点

  • 检查发送给 provider 的 JSON 字段是否正确。
  • 检查 provider 成功响应是否映射为 PredictionResult。
  • 检查 provider HTTP 503 是否映射为稳定错误。
  • 检查 provider 返回非法 JSON 是否映射为稳定错误。

常见坑

  • 把 provider 原始 JSON 直接返回给前端。
  • 只检查 HTTP 200，不校验业务字段。
  • 在 adapter 里吞掉 provider 错误。
  • 测试只覆盖成功路径。

面试短答

Provider prediction adapter 负责把本项目输入转换成 provider 请求 JSON，并把 provider 响应 JSON 校验后转换成内部 PredictionResult。它屏蔽 provider 字段形状和 HTTPX 异常，上层只依赖稳定结果和 AIClientError。

来源：FastAPI Request Body、 FastAPI Response Model、 Pydantic Models、 HTTPX QuickStart、 HTTPX Exceptions。
