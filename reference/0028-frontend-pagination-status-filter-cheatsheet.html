Reference 0028

前端分页与状态筛选速查

控件改变前端状态，前端状态生成查询参数，响应信封再更新控件。

状态对象

const historyState = {
  status: "",
  limit: 5,
  offset: 0,
  count: 0,
  itemsLength: 0,
};

构造 URL

const params = new URLSearchParams({
  limit: String(historyState.limit),
  offset: String(historyState.offset),
});

if (historyState.status) {
  params.set("status", historyState.status);
}

筛选按钮

<button data-status="failed" type="button">Failed</button>

historyState.status = button.dataset.status;
historyState.offset = 0;

分页规则

previous_offset = max(offset - limit, 0)
has_next = offset + itemsLength < count
next_offset = offset + limit

常见坑

  • 筛选变化后不重置 offset。
  • 手写查询字符串，导致空参数或拼接错误。
  • 不用 count 判断下一页。
  • 按钮选中态和真实请求状态不同步。
来源：MDN URLSearchParams、 MDN HTMLElement.dataset、 FastAPI Query Parameters。
