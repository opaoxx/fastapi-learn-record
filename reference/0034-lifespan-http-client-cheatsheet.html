Reference 0034

Lifespan HTTP Client 速查

应用启动时创建长期资源，应用关闭时清理资源。

核心心智模型

lifespan startup
  -> create AsyncClient
  -> store in app.state
  -> serve requests
lifespan shutdown
  -> await client.aclose()

基本写法

@asynccontextmanager
async def lifespan(app: FastAPI):
    app.state.provider_http_client = httpx.AsyncClient(
        timeout=10.0,
        trust_env=False,
    )
    try:
        yield
    finally:
        await app.state.provider_http_client.aclose()

观察端点

GET /provider/http-client

{
  "provider": "demo",
  "timeout_seconds": 10.0,
  "client_ready": true
}

适合放进 app.state

  • HTTP client
  • 数据库连接池
  • 缓存客户端
  • 模型加载结果
  • 消息队列连接

常见坑

  • 每个请求都新建 AsyncClient。
  • 创建 client 后忘记关闭。
  • 把请求级数据放进 app.state。
  • 忽略环境代理对 HTTPX 的影响。
来源：FastAPI Lifespan Events、 HTTPX Clients、 Starlette app.state。
