Reference 0041

Prediction Service Orchestration 速查

router 负责 HTTP 翻译，service 负责预测业务编排。

分层心智模型

router
  -> HTTP request / dependencies / headers / HTTPException

service
  -> backend selection / adapter calls / elapsed time / internal errors

成功结果

@dataclass(frozen=True)
class PredictionServiceResult:
    prediction: PredictionResult
    backend: PredictionBackend
    elapsed_ms: float

失败结果

class PredictionServiceError(Exception):
    message: str
    error_code: str
    backend: PredictionBackend
    elapsed_ms: float

service 函数

result = await run_prediction(
    text=payload.text,
    mode=payload.mode,
    settings=settings,
    ai_client=ai_client,
    provider_http_client=provider_http_client,
)

router 翻译

try:
    result = await run_prediction(...)
except PredictionServiceError as exc:
    raise HTTPException(
        status_code=503,
        detail={
            "error_code": exc.error_code,
            "message": exc.message,
        },
        headers=build_prediction_headers(...),
    ) from exc

测试分工

  • service 测试：后端选择、耗时、内部错误包装。
  • router 测试：依赖注入、HTTP 状态码、headers、JSON 响应。

常见坑

  • router 里塞满业务分支。
  • service 直接抛 HTTPException。
  • 错误包装时丢掉 backend 或 elapsed_ms。
  • 只有 HTTP 测试，没有 service 单元测试。

面试短答

我会把预测业务编排放到 service 层，让它返回 PredictionServiceResult 或抛 PredictionServiceError。router 只把这些内部结果翻译成 HTTP 响应、headers、日志和 HTTPException。这样路由更薄，业务逻辑更容易单独测试。
