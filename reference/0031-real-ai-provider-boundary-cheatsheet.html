Reference 0031

真实 AI Provider 边界速查

router 依赖项目自己的 AIClient 协议，provider adapter 负责处理外部世界。

核心心智模型

FastAPI router
  -> Depends(get_ai_client)
  -> AIClient Protocol
  -> provider adapter
  -> external AI service

协议形状

class AIClient(Protocol):
    def predict_sentiment(self, text: str, mode: str) -> PredictionResult:
        ...

    def summarize(self, text: str) -> str:
        ...

配置

FIRST_API_AI_PROVIDER="demo"
FIRST_API_AI_TIMEOUT_SECONDS="10"
FIRST_API_AI_MAX_ATTEMPTS="1"

边界层职责

  • 把内部请求转换成 provider 请求。
  • 把 provider 响应转换成内部结果。
  • 把 timeout、限流、5xx、格式错误映射成稳定错误。
  • 集中处理 provider 配置。
  • 让测试可以使用替身对象。

测试替换

app.dependency_overrides[get_ai_client] = override_ai_client

try:
    response = client.post("/predict", json=payload)
finally:
    app.dependency_overrides.clear()

常见坑

  • 在 router 里直接调用 provider SDK。
  • 没有 timeout，导致请求长期挂起。
  • 把 provider 原始错误直接暴露给前端。
  • 自动化测试真的调用外部模型。
来源：FastAPI Dependencies、 FastAPI Testing Dependencies with Overrides、 Python Protocols、 HTTPX Timeouts。
