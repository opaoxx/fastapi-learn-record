Reference 0045

Retry Jitter 速查

jitter 用随机扰动打散多个客户端的重试时间。

心智模型

backoff = slow down retries
jitter  = spread out retries

本项目公式

jitter_delay = retry_jitter_seconds * random_fraction
delay = min(exponential_delay + jitter_delay, max_delay)

配置

FIRST_API_AI_RETRY_BASE_DELAY_SECONDS="0.1"
FIRST_API_AI_RETRY_MAX_DELAY_SECONDS="2.0"
FIRST_API_AI_RETRY_JITTER_SECONDS="0.1"

核心函数

def calculate_retry_jitter(config, random_fraction=random.random):
    jitter_seconds = max(config.retry_jitter_seconds, 0.0)
    if jitter_seconds == 0:
        return 0.0

    fraction = min(max(random_fraction(), 0.0), 1.0)
    return jitter_seconds * fraction

边界规则

  • 只给本地 backoff 加 jitter。
  • 有效 Retry-After 不额外加 jitter。
  • jitter 后仍然受 retry_max_delay_seconds 截断。
  • 测试中注入固定 random_fraction。

测试例子

delay = calculate_retry_delay(
    attempt=2,
    config=config,
    random_fraction=lambda: 0.4,
)

assert delay == 2.2

常见坑

  • 真实随机数进入断言测试。
  • jitter 后不做 max delay 截断。
  • 给 Retry-After 再随机加等待。
  • 把 random 当安全随机数使用。

面试短答

backoff 能放慢重试，但多个客户端仍可能同步。jitter 给本地 backoff 加小随机扰动，把请求峰值打散。我会把 jitter 做成配置，并把随机函数注入测试，让随机逻辑可重复验证。
