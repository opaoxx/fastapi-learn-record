Reference 0059

Provider Metrics Naming Contract 速查

metric name、HELP 文本和 label names 是监控 API，会被查询、Dashboard、告警和文档依赖。

一句话模型

指标命名不是内部细节；一旦导出，它就是可查询的公共契约。

当前 metric

PROVIDER_PREDICTION_METRIC_NAME = "provider_prediction_total"
PROVIDER_PREDICTION_METRIC_HELP = (
    "Provider prediction outcomes recorded by the teaching metrics counter."
)

命名拆解

片段 含义
provider 外部 provider adapter 路径。
prediction 被测量对象是 prediction 请求结果。
total 累计 counter，总次数。

固定 label keys

[
    "operation",
    "outcome",
    "failure_category",
    "error_code",
    "status_code",
]

样本 label set

{
    "operation": "prediction",
    "outcome": "retry_exhausted",
    "failure_category": "retryable_http_status",
    "error_code": "ai_provider_http_error",
    "status_code": "503",
}

契约测试

assert PROVIDER_PREDICTION_METRIC_NAME == "provider_prediction_total"
assert PROVIDER_PREDICTION_METRIC_NAME.endswith("_total")
assert list(labels.as_dict()) == [
    "operation",
    "outcome",
    "failure_category",
    "error_code",
    "status_code",
]

低基数守则

  • 可以放：outcome、failure category、稳定 error code、HTTP status code。
  • 不要放：用户输入文本、用户 ID、邮箱、request id、完整 URL。
  • 原因：每组 label key/value 都会形成独立时间序列。

运行命令

.\.venv\Scripts\python.exe -m pytest tests\test_provider_http.py -q

常见错误

  • 随手改 metric name。
  • 把 label name 放进 metric name。
  • 把高基数用户数据放进 label value。
  • 多个 metric name 表达同一件事。
  • 只保护文本输出，不保护命名常量。

面试短答

metric name 和 label names 是监控 API。它们要稳定、低基数、便于聚合，并由测试保护，因为 Dashboard、告警和排障查询会依赖这些名字。

来源：Prometheus metric and label naming、 Prometheus data model、 Prometheus instrumentation practices。
