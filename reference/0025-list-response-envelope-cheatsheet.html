Reference 0025

列表响应信封速查

裸数组只表达当前有什么；响应信封表达当前页在整个结果集里的位置。

响应模型

class SummaryTaskListResponse(BaseModel):
    items: list[SummaryTaskRead]
    count: int
    limit: int
    offset: int

响应示例

{
  "items": [],
  "count": 0,
  "limit": 20,
  "offset": 0
}

字段含义

  • items：当前页业务数据。
  • count：当前筛选条件下的总记录数。
  • limit：本次最多返回多少条。
  • offset：本次跳过多少条。

调用方变化

# 旧：裸数组
response[0]

# 新：响应信封
response["items"][0]

常见坑

  • 前端仍按数组读取响应。
  • 把 count 当成当前页长度。
  • 只改后端，不改测试、文档和前端调用方。
  • 字段名不稳定，导致调用方难维护。
来源：FastAPI Response Model、 FastAPI Query Parameters。
