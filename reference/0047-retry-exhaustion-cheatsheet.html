Reference 0047

Retry Exhaustion 速查

Retry exhaustion 表示可重试错误已经用完所有 attempt，必须停止并向上抛出失败。

心智模型

retry budget = max_attempts
scheduled retry = failed, but can try again
retry exhausted = failed, and no attempts left
fail fast = should not retry at all

日志事件

provider_prediction_retry_scheduled
provider_prediction_retry_exhausted

Exhausted 字段

  • attempt：最终失败发生在第几次尝试。
  • max_attempts：最多允许尝试几次。
  • retry_reason：timeout、http_status 或 request_error。
  • status_code：HTTP 状态失败时记录。
  • retry_after：最后一次 HTTP 响应里的 Retry-After。
  • error_code：上抛给 service/router 的稳定错误码。

核心规则

if retryable and attempt < max_attempts:
    log scheduled retry
    sleep
    continue

if retryable and attempt == max_attempts:
    log retry exhausted
    raise AIClientError

if not retryable:
    raise AIClientError

代码形状

logger.warning(
    "provider_prediction_retry_exhausted",
    extra={
        "attempt": attempt,
        "max_attempts": config.max_attempts,
        "retry_reason": retry_reason,
        "status_code": status_code,
        "error_code": error_code,
    },
)

caplog 断言

caplog.set_level(logging.WARNING, logger="first_api.services.provider_http")

assert exhausted_records[0].attempt == 3
assert exhausted_records[0].max_attempts == 3
assert exhausted_records[0].status_code == 503
assert exhausted_records[0].error_code == "ai_provider_http_error"

常见边界

  • 503 连续失败到最后一次：记录 exhausted。
  • timeout 连续失败到最后一次：记录 exhausted。
  • request error 连续失败到最后一次：记录 exhausted。
  • 400：快速失败，不记录 exhausted。
  • 非法响应体：快速失败，不记录 exhausted。

面试短答

我会把 max_attempts 看作 retry budget。预算内的可重试错误记录 scheduled retry；最后一次仍失败时记录 retry exhausted warning 并抛出稳定错误码。不可重试错误快速失败，不误标为 exhausted。

来源：Python logging、 pytest logging、 HTTPX exceptions。
