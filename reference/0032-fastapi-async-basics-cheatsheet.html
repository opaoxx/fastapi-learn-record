Reference 0032

FastAPI async 基础速查

async 提升等待型 I/O 的并发效率，不会让 CPU 计算自动变快。

核心心智模型

async def = 创建可暂停的协程函数
await = 等待并让出执行权
event loop = 调度可运行的任务
blocking call = 占住线程，不能让出

教学端点

GET /async/wait?delay=0.2

{
  "status": "ok",
  "delay_seconds": 0.2,
  "message": "The endpoint awaited without doing CPU work."
}

基本写法

@router.get("/async/wait")
async def async_wait(delay: Annotated[float, Query(ge=0, le=1)] = 0.05):
    await asyncio.sleep(delay)
    return {"status": "ok"}

选择规则

场景 建议
可 await 的 HTTP / DB / I/O async def
同步库和同步数据库 普通 def 通常更清楚
CPU 密集计算 线程、进程、队列或专门服务

常见坑

  • 以为 async 会让 CPU 计算变快。
  • 在 async def 里调用 time.sleep。
  • 忘记 await，拿到协程对象。
  • 把同步 SQLModel 代码直接改成 async。
