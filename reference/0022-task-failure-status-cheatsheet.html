Reference 0022

任务失败状态速查

后台任务的最终结果写在任务记录里，而不是写在提交接口的 HTTP 状态码里。

状态流转

queued -> running -> completed
queued -> running -> failed

失败响应形状

{
  "status": "failed",
  "result": null,
  "error": "summary_model_timeout: The summary model timed out."
}

处理规则

  • 提交任务成功：202 Accepted。
  • 任务还没完成：查询接口返回 queued 或 running。
  • 任务完成：查询接口返回 completed 和 result。
  • 任务失败：查询接口返回 failed 和 error。
来源：FastAPI Background Tasks、 MDN 202 Accepted。
