Reference 0063

Provider Metrics Scenario Fixtures 速查

固定样本用于练 runbook 判断，不代表当前运行时指标。

一句话模型

scenario fixture 固定“看到什么”，runbook 训练“下一步查什么”。

核心函数

build_provider_metrics_runbook_scenario_samples()
  • 返回 list[dict[str, object]]，shape 与 metrics snapshot 兼容。
  • 复用 build_provider_metric_labels()，避免手写 label contract。
  • 不调用 record_provider_metric()，不会污染全局 counter。

四类样本

outcomefailure_categorystatus_codecount判断重点
successnonenone3成功基线仍存在。
retry_scheduledretryable_http_status5032安排过重试，不等于最终失败。
retry_exhaustedretryable_http_status5031最大尝试次数已耗尽。
fail_fastnon_retryable_http_status4001不要盲目增加重试次数。

测试命令

.\.venv\Scripts\python.exe -m pytest tests\test_provider_http.py -q
.\.venv\Scripts\python.exe -m pytest tests\test_course_docs_contract.py -q

记忆要点

  • fixture 是练习数据，不是运行数据。
  • 固定样本必须覆盖 success、retry_scheduled、retry_exhausted、fail_fast。
  • 样本可以传给 render_provider_metrics_prometheus_text() 练习 text export。
  • 教学 fixture 不能直接升级成生产监控模拟器。
