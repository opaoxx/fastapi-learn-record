Reference 0040

Prediction Observability 速查

用响应头和日志记录预测 backend、source、elapsed_ms 和 error_code，不改变 JSON 响应体。

成功响应头

X-Prediction-Backend: demo
X-Prediction-Source: demo-ai-client
X-Prediction-Elapsed-Ms: 1.23

失败响应头

X-Prediction-Backend: provider
X-Prediction-Elapsed-Ms: 10.42
X-Prediction-Error-Code: ai_provider_http_error

耗时测量

started_at = perf_counter()
elapsed_ms = (perf_counter() - started_at) * 1000

设置 headers

def set_prediction_headers(
    response: Response,
    backend: str,
    elapsed_ms: float,
    source: str | None = None,
) -> None:
    response.headers["X-Prediction-Backend"] = backend
    response.headers["X-Prediction-Elapsed-Ms"] = f"{elapsed_ms:.2f}"
    if source is not None:
        response.headers["X-Prediction-Source"] = source

成功日志

logger.info(
    "prediction_completed",
    extra={
        "prediction_backend": backend,
        "prediction_source": prediction.source,
        "prediction_elapsed_ms": round(elapsed_ms, 2),
    },
)

失败日志

logger.warning(
    "prediction_failed",
    extra={
        "prediction_backend": backend,
        "prediction_elapsed_ms": round(elapsed_ms, 2),
        "prediction_error_code": exc.error_code,
    },
)

测试断言

caplog.set_level(logging.INFO, logger=PREDICTIONS_LOGGER)

assert response.headers["x-prediction-backend"] == "demo"
assert float(response.headers["x-prediction-elapsed-ms"]) >= 0

log_record = next(
    record for record in caplog.records
    if record.message == "prediction_completed"
)
assert log_record.prediction_backend == "demo"

设计规则

  • 调试元信息优先放 header 或日志，不轻易改 JSON body。
  • 成功和失败都要记录耗时。
  • 失败日志至少包含 backend 和 error_code。
  • 不要把完整用户输入文本写进日志。
来源：Python logging、 Python time、 FastAPI Response Headers、 pytest Logging。
