Reference 0052

Provider Metrics Endpoint 速查

metrics endpoint 是只读观测接口：service 层记录 provider prediction 事件，router 读取 counter snapshot 并输出稳定 JSON。

心智模型

business event happens
service records metric labels
counter aggregates same label set
GET /provider/metrics reads snapshot

Endpoint

GET /provider/metrics

只读，不调用 provider，不改变业务状态。

响应形状

[
  {
    "labels": {
      "operation": "prediction",
      "outcome": "success",
      "failure_category": "none",
      "error_code": "none",
      "status_code": "none"
    },
    "count": 1
  }
]

核心代码

def record_provider_metric(labels: ProviderMetricLabels, amount: int = 1) -> None:
    provider_prediction_metrics.increment(labels, amount=amount)

def get_provider_metrics_snapshot() -> list[dict[str, object]]:
    return provider_prediction_metrics.snapshot()

Router 形状

@router.get(
    "/provider/metrics",
    response_model=list[ProviderMetricSampleResponse],
)
async def get_provider_metrics() -> list[dict[str, object]]:
    return get_provider_metrics_snapshot()

Outcome 来源

outcome 什么时候记录
success provider 返回合法 prediction body
retry_scheduled timeout、request error 或可重试 HTTP 状态准备重试
retry_exhausted 可重试失败用完最大尝试次数
fail_fast 非重试 HTTP 状态或非法 provider 响应

测试重点

reset_provider_metrics()
client.post("/provider/predict", json=payload)
response = client.get("/provider/metrics")

assert response.json()[0]["labels"]["outcome"] == "success"
assert response.json()[0]["count"] == 1

常见错误

  • 读取 metrics 时又调用 provider。
  • 测试之间不重置进程内 counter。
  • 把 request text、request id 放进 labels。
  • 把教学用 in-process counter 当成生产监控。
  • 在 router 里堆统计规则，导致 HTTP 层过重。

边界

  • 适合教学 metrics 形状、response model 和测试隔离。
  • 不解决多进程、多实例、重启后丢失和标准抓取格式。
  • 生产项目应接入 Prometheus、OpenTelemetry 或云监控系统。

面试短答

metrics endpoint 应该是只读观测接口。provider prediction 的成功、重试、耗尽和 fail fast 在 service 层记录成低基数 labels，router 只返回 snapshot，并用 response model 固定 JSON 契约。当前 in-process counter 是教学实现，生产环境需要外部 metrics 后端聚合。

来源：FastAPI response model、 Prometheus exposition formats、 OpenTelemetry Metrics。
