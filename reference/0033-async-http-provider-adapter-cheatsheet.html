Reference 0033

异步 HTTP Provider Adapter 速查

外部 HTTP 细节留在 adapter，项目内部只看稳定结果和稳定错误。

核心心智模型

httpx.AsyncClient
  -> provider HTTP response / exception
  -> provider adapter
  -> ProviderHealthResult or AIClientError

成功路径

response = await active_client.get(url)
response.raise_for_status()

return ProviderHealthResult(
    ok=True,
    provider=config.provider,
    status_code=response.status_code,
    message="The AI provider responded successfully.",
)

错误映射

外部异常 内部错误码
httpx.TimeoutException ai_provider_timeout
httpx.HTTPStatusError ai_provider_http_error
httpx.RequestError ai_provider_request_error

MockTransport

def handler(request: httpx.Request) -> httpx.Response:
    return httpx.Response(status_code=200, json={"status": "ok"}, request=request)

transport = httpx.MockTransport(handler)
client = httpx.AsyncClient(transport=transport)

常见坑

  • 测试访问真实 provider。
  • 忘记 raise_for_status()。
  • 对外暴露 HTTPX 原始异常。
  • 关闭不属于自己的 client。
来源：HTTPX Async Support、 HTTPX Timeouts、 HTTPX Exceptions。
