Reference 0051

Provider Metrics Counters 速查

Counter 用来记录事件发生次数；相同 label set 的事件累加到同一条计数。

心智模型

counter = only goes up
same labels = same time series
different labels = different count

代码形状

@dataclass
class ProviderMetricsCounter:
    _counts: Counter[ProviderMetricLabels] = field(default_factory=Counter)

递增规则

counter.increment(success_labels)
counter.increment(success_labels)
counter.increment(fail_fast_labels)

Snapshot 形状

[
  {"labels": fail_fast_labels.as_dict(), "count": 1},
  {"labels": success_labels.as_dict(), "count": 2},
]

边界

  • 这是教学用进程内 counter。
  • 进程重启后数据会消失。
  • 多进程、多实例不会自动聚合。
  • 真实生产需要 Prometheus、OpenTelemetry 或云监控等后端。

测试重点

assert counter.snapshot()[1]["count"] == 2

try:
    counter.increment(labels, amount=0)
except ValueError:
    ...

常见错误

  • 用 counter 表达会上下变化的值。
  • 把进程内 counter 当生产方案。
  • 把 request id 放进 label。
  • snapshot 顺序不稳定导致测试脆弱。

面试短答

Counter 适合统计事件次数，只递增不递减。provider prediction 可以按 outcome、failure_category、error_code、status_code 等低基数 label 聚合；单次请求上下文仍然放在结构化日志里。

来源：Prometheus metric types、 OpenTelemetry Metrics、 Python Counter。
