Provider Metrics 文档入口

这一页把 provider prediction metrics 的端点、指标名、边界说明、重命名审查点和测试入口收在一起。它是课程里的小型文档索引，不是生产监控平台配置。

1. 先看哪个端点

入口 用途 边界
GET /provider/metrics 读取教学用 in-process provider prediction metric counter snapshot。 适合学习响应形状和 label 聚合，不适合生产多实例聚合。
GET /provider/metrics/prometheus 读取 Prometheus text-style rendering。 这是 educational export shape, not a production multi-instance metrics pipeline。

2. 指标契约

当前导出的教学指标名是 provider_prediction_total，HELP 文本是 Provider prediction outcomes recorded by the teaching metrics counter.

指标 label keys 是 operation、outcome、failure_category、error_code 和 status_code。这些字段必须保持低基数，不要放入用户输入文本、请求 ID、完整异常堆栈或高变化 provider message。

如果未来真的要改指标名或 label name，先看 get_provider_prediction_metric_contract，再设计兼容窗口；不要把 metrics rename 当成普通变量重命名。

  • tests/test_provider_http.py：保护 provider metrics label、counter、snapshot、Prometheus text rendering 和 rename-sensitive contract。
  • tests/test_openapi_contract.py：保护 provider metrics endpoint 在 OpenAPI 里的生产边界说明。
  • tests/test_course_docs_contract.py：保护本页作为文档入口时必须包含的端点、指标和测试链接。

5. 排障入口

如果问题不是“文档在哪里”，而是“看到指标变化后下一步查什么”，先打开 Provider Metrics Runbook。

6. 修改前检查清单

  1. 改 endpoint 文档前，先确认 OpenAPI contract test 是否需要同步更新。
  2. 改 metric name、HELP 或 label name 前，先确认是否需要兼容窗口和消费者迁移。
  3. 改文本导出前，同时检查 JSON snapshot、Prometheus text renderer、endpoint 测试和本页。
  4. 改课程讲义前，确认本页仍能把 0058 到 0064 的学习路线串起来。
